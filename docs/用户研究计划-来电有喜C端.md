# 来电有喜 · C 端用户研究计划（需求与痛点综合探索）

> 文档类型：User Research Study Plan（混合方法 / Mixed-Methods）
> 研究主体：终端消费者（C 端买家）
> 研究目标：需求发现 + 现有核心流程痛点诊断
> 状态：方案已就绪，待输入真实研究材料后执行分析
> 日期：2026-07-08

---

## 0. 一句话定位

来电有喜是一个**武侠国潮风的本地社交电商小程序**：用户在合作门店购物（配送/自提），可领红包/实物礼品（"来电有喜"活动）、按情绪标签发现商品、攒积分升段位（江湖散修→掌门），并通过推广码赚二级分销佣金。本研究聚焦**终端消费者**，用科学方法同时回答"用户真正想要什么"和"现有流程哪里卡住了"。

## 1. 研究背景与产品上下文

### 1.1 C 端核心功能面（来自代码与文档梳理）
| 模块 | 关键页面/能力 | 潜在研究关注点 |
|------|--------------|---------------|
| 电商购物 | `index` 首页、`explore` 发现、`search` 搜索、`product` 商品详情、`cart` 购物车、`payment` 支付、`order-center` 订单 | 首单转化、搜索/发现效率、支付双通道（微信支付 / gold_beans 金豆）认知 |
| 营销活动 | `campaign-claim` 领红包/礼品、`reward-shop` 奖品商城、`coupon` 优惠券 | 领取→核销链路、地理/门店限制、核销门槛造成的流失 |
| 情绪购物（创新） | `emotion-check` 情绪检测、`emotion-detail`、`emotion-bill` 情绪账单、`emotion-badges` 情绪勋章 | 用户是否真把"情绪"当购物决策入口，还是噱头 |
| 积分与成长 | `user` 个人中心、`points-rules` 积分规则、`rank-rules` 段位规则 | 成长体系是否驱动留存，还是无感 |
| 推广分佣 | `my-promotion` 我的推广、`commission-detail` 佣金明细、`withdraw` 提现、`my-referrals` 我的推荐 | 被"永久锁定"给某推广员的感知与接受度（体验+合规风险） |
| 内容与社交 | `content-center` 内容中心、`ugc-publish` 发布、`article-detail`、`favorites` 收藏、`footprint` 足迹 | UGC 驱动力、内容→消费转化 |

### 1.2 来自产品文档的"待确认事项" → 已翻译为研究问题
产品文档（分佣积分机制设计、段位方案等）留下多处未决决策，这些正是用户研究要回答的：

- **RQ-产品-1**：用户对"让利/分佣"机制是否**有感知**？感知后是增强信任还是引起疑虑？
- **RQ-产品-2**：C 端用户对被"**永久锁定**"给某推广员（首次扫码归属制）的接受度与情绪？是否存在被操控感、合规顾虑？
- **RQ-产品-3**：**情绪购物**是否解决真实需求，还是自嗨式功能？用户决策时真的参考情绪标签/勋章吗？
- **RQ-产品-4**：**积分 + 段位**成长体系对用户留存/复购的的真实驱动力有多大？12 个月积分有效期是否合理？
- **RQ-产品-5**：**红包/活动领取→核销**全链路的真实顺畅度，核销门槛（到店、地理、时间）是否造成核心痛点？

## 2. 研究问题（Research Questions）

### 2.1 一级研究问题（主目标）
- **RQ1（需求发现）**：终端消费者在"本地生活消费 + 省钱/赚钱 + 情绪满足"场景下的真实需求、动机与生活语境是什么？有哪些未被满足的机会？
- **RQ2（痛点诊断）**：在核心流程（首单下单、领红包并核销、情绪发现、积分查看与抵扣、段位进度感知、推广赚佣金）中，用户在哪里流失、困惑、受挫？
- **RQ3（价值验证）**：情绪购物、永久锁定、让利分佣等创新/争议机制，用户实际价值感知如何？

### 2.2 二级拆解（用于访谈提纲与问卷量表）
- RQ1-a：用户如何发现/选择合作门店与商品？信息入口偏好？
- RQ1-b：用户使用小程序的**核心场景与频次**（偶发领红包 vs 日常复购 vs 推广赚钱）？
- RQ2-a：支付环节（微信支付 vs 金豆）是否存在认知与操作负担？
- RQ2-b：领取红包后"去哪核销、怎么核销"是否构成断点？
- RQ3-a：情绪标签/勋章对"逛"的体验与购买决策的影响程度？
- RQ3-b：用户是否意识到自己被某推广员锁定？态度如何？

## 3. 研究方法论（Mixed-Methods，分四阶段）

> **方法选择原则**：定性先探（发现未知），定量后证（规模校验），行为数据补盲（真实而非自陈），可用性测试定位具体卡点。四路数据三角验证，避免单一方法偏差。

### Phase 1 — 探索性定性访谈（Qualitative Discovery）
- **目的**：回答 RQ1/RQ3，发现未知需求与态度，生成假设。
- **方法**：半结构化深度访谈（45–60 min），think-aloud + 情境回忆。
- **样本**：n = 12–15 名 C 端用户。
- **样本量依据**：Guest 等（2006）表明同质群体约 6 次访谈覆盖主要主题，12+ 可达主题饱和；跨多样人群取 15 以覆盖异质性。
- **产出**：访谈转录、初始编码、假设清单。

### Phase 2 — 定量问卷（Quantitative Survey）
- **目的**：校验 Phase 1 假设的规模与分布，回答 RQ2/RQ3 的广度量级。
- **方法**：结构化在线问卷（李克特量表 + 多选 + 人口统计），小程序内弹窗 + 社群投放。
- **样本**：n = 300–400 有效回收。
- **样本量依据**：95% 置信度、±5% 误差下比例估计需 n≈385；若做亚组（新/老用户、高/低活跃）对比，建议 ≥400 并设配额。
- **产出**：需求优先级矩阵、痛点发生率、分群画像。

### Phase 3 — 行为数据分析（Behavioral Analytics）
- **目的**：用真实行为（非自陈）验证流失与转化假设，回答 RQ2。
- **数据来源**：Supabase 导出（建议表：`profiles`、`orders`、`user_campaign_claims`、`footprints`、`favorites`、`points_logs`、`cart_items`）。
- **分析**：转化漏斗（首页→商品→加购→支付→完成）、活动领取→核销率、留存曲线（D1/D7/D30）、分群对比（新客 vs 回流 vs 推广者）。
- **产出**：量化漏斗断点、高流失环节排序。

### Phase 4 — 可用性测试（Usability Testing）
- **目的**：定位核心流程具体卡点与可改问题，回答 RQ2 的"为什么卡"。
- **方法**：任务型测试（moderated，录屏 + 出声思考），n = 8–10 人。
- **样本量依据**：Nielsen（2000）表明 5 人可发现约 80% 可用性问题，8–10 人覆盖更稳且含多样性。
- **核心任务**：① 完成首单（含选门店/配送/支付）② 领取并核销一个红包/礼品 ③ 用情绪标签发现一件商品 ④ 查看积分并抵扣 ⑤ 查看段位进度 ⑥（如适用）生成推广码并理解锁定关系。
- **产出**：任务完成率、耗时、错误数、SUS 量表分、具体痛点清单。

## 4. 参与者标准（Participant Criteria）

### 4.1 主要用户
终端消费者：近 90 天在来电有喜有过任意行为（浏览/领红包/下单/推广）的微信用户。

### 4.2 分层与配额（保证多样性，防偏差）
| 维度 | 分层 | 目标占比 |
|------|------|---------|
| 活跃度 | 新用户（<30天）/ 活跃用户 / 沉睡用户 | 各 ~1/3 |
| 行为 | 纯消费者 / 领红包党 / 推广者（兼用户） | 混合覆盖 |
| 年龄 | 18–25 / 26–35 / 36–45 / 46+ | 按产品实际分布 |
| 地域 | 一二线 / 下沉市场 | 混合 |
| **无障碍** | **含老花/低视力、运动障碍、老年用户各 1–2 名** | **必含（默认要求）** |

### 4.3 样本量统计依据
- 定性：12–15（主题饱和）。
- 定量：385（±5% MoE @95%）；亚组对比加配额至 400+。
- 可用性：8–10（问题检出率 ~80–100%）。

### 4.4 招募与筛选
- **渠道**：小程序内调研弹窗、微信社群、合作门店地推、朋友圈招募。
- **激励**：红包/积分补偿（符合伦理，不影响回答中立性）。
- **筛选问卷**：过滤非目标人群、平衡配额、排除内部人员。
- **偏差防控**：避免仅从"推广者社群"招募（会系统性高估分销价值）；定性阶段覆盖沉默大多数。

## 5. 研究协议（Study Protocol）

### 5.1 定性访谈提纲（节选）
1. 回顾：上次在小程序里做了什么？最开心/最烦的一次经历？
2. 发现：你一般怎么找到想买的东西/想领的红包？
3. 情绪：看到"情绪标签/勋章"时，你会因此多看或购买吗？
4. 锁定：你是否知道自己被某推广员"绑定"？知道后什么感受？
5. 价值：积分、段位对你继续用有多大影响？
6. 痛点：领了红包后，去核销顺利吗？哪里卡住？
7. 理想：如果有个"完美版来电有喜"，你最想要什么？

### 5.2 问卷量表（节选）
- 需求优先级：多选 + 排序（省钱/赚钱/便利/情绪满足/社交/信任）。
- 痛点发生率：近 90 天是否遇到 [支付困惑/核销失败/找不到商品/不理解锁定/积分无用]（是/否 + 频率）。
- NPS / 满意度（1–5）。
- 人口统计 + 活跃度分层。

### 5.3 可用性任务场景（示例）
- **任务① 首单**："你想买一杯奶茶，找到附近门店下单并支付。" 成功标准：完成支付。指标：完成率、耗时、错误。
- **任务② 核销**："你领了一个红包，去对应门店核销使用。" 成功标准：核销成功。重点观察：是否知道去哪核销。
- **任务③ 情绪发现**："今天有点低落，用情绪功能找件能让自己开心的小东西。" 重点观察：是否理解情绪入口。
- **任务④ 积分抵扣**："结算时用积分抵一部分钱。" 重点观察：抵扣规则认知。

### 5.4 时间线（4–6 周，可并行）

|------|------|------|
| W1 | 招募 + 访谈启动 | 访谈转录 |

## 6. 分析框架（Analysis & Synthesis Framework）
> 这是"当你把材料发我后，我直接执行"的处理流程。方法学对齐 Braun & Clarke 主题分析 + 描述/推断统计 + 可用性指标。

### 6.1 定性（访谈/笔记）
1. 转录与熟悉数据
2. 初始编码（open coding，逐句标签）
3. 主题搜索（affinity mapping，归并成主题）
4. 主题复核与命名
5. 跨受访者模式识别（出现频次 + 强度 + 典型引语）
6. 输出：主题地图 + 引语证据库

### 6.2 定量（问卷 / 行为数据）
- **描述统计**：需求占比、痛点发生率、NPS、满意度均值。
- **推断统计**：卡方 / t 检验做亚组差异（新 vs 老、高 vs 低活跃），α=0.05；效应量报告。
- **分群**：按行为/动机做聚类，识别用户类型。
- **漏斗**：各步骤转化率、断点排序（行为数据）。

### 6.3 三角验证矩阵
每个核心结论需 ≥2 个独立数据源支撑；冲突结论标注并单列（不强行一致）。

### 6.4 综合产出
- **用户 Persona**（2–4 个，含真实引语、目标、痛点、场景）。
- **用户旅程图（现状）**：触点 → 行为 → 情绪曲线 → 痛点 → 机会。
- **可用性指标表**：任务完成率/耗时/错误/SUS。
- **优先级建议**：Impact × Effort 矩阵，标注"立即改 / 下季度 / 长期"。

### 6.5 无障碍与包容性发现
单独维度报告：老年/低视力/运动障碍用户在核心流程中的障碍，给出 WCAG 2.1 AA 级别建议。

## 8. 数据伦理（Ethical Practices）
- 知情同意：录音/数据使用需书面/电子授权，可随时退出。
- 隐私保护：PII 脱敏，分级存储，最小必要原则。
- 包容性招募：跨年龄/能力/地域，避免抽样偏差。
- 客观呈现：不带确认偏差，负面结论同样报告。
- 合规提示：涉及"永久锁定/二级分销"等敏感机制，研究中记录用户真实态度，供法务/合规参考。
