## 3️⃣ 订单/分佣/积分逻辑问题 ⚠️

### 问题 1：创建门店时未生成 `short_code`

**位置：** `admin-web/src/api/admin.ts` 第 158-169 行

**问题：**
```typescript
// 创建门店记录时，没有生成 short_code
const { error: storeError } = await supabase
  .from('stores')
  .insert({
    owner_id: app.user_id,
    name: app.store_name,
    description: app.description || null,
    phone: app.contact_phone || null,
    category: app.business_type || '其他',
    is_active: true,
    rating: 0,
  })
```

**影响：** 门店二维码无法生成（需要 `short_code`）

**修复方案：**
```typescript
// 生成短码（8位随机字母数字）
const shortCode = Math.random().toString(36).substring(2, 10).toUpperCase()

await supabase.from('stores').insert({
  owner_id: app.user_id,
  name: app.store_name,
  short_code: shortCode,  // ← 新增
  // ... 其他字段
})
```

### 问题 2：驳回后用户无法重新申请

**位置：** `src/pages/merchant-apply/index.tsx` 第 94-98 行

**问题：**
```tsx
{existing.status === 'rejected' && (
  <>
    <div className="i-mdi-close-circle text-8xl text-destructive" />
    <p className="text-2xl font-bold text-destructive">审核未通过</p>
    <p className="text-xl text-muted-foreground text-center">原因：{existing.reject_reason || '申请信息不符合要求'}</p>
    <button type="button"
      className="flex items-center justify-center leading-none rounded-2xl bg-primary"
      onClick={() => setExisting(null)}>  // ← 只是清空 existing，未清空表单
      <div className="py-3 px-8 text-xl text-white font-bold">重新申请</div>
    </button>
  </>
)}
```

**影响：** 点击"重新申请"后，表单数据还在，用户可能提交旧数据

**修复方案：**
```tsx
onClick={() => {
  setExisting(null)
  setStoreName('')
  setContactName('')
  setContactPhone('')
  setBusinessType('餐饮')
  setDescription('')
}}
```

## 5️⃣ 缓存策略问题 ⚠️

### 问题：Supabase 客户端未配置缓存

**位置：** `src/client/supabase.ts`

**问题：**
```typescript
const realSupabase = createClient(supabaseUrl, supabaseAnonKey, {
  global: { fetch: customFetch },
  auth: {
    storageKey: `${appId}-auth-token`,
    storage: taroStorage,
  }
  // ← 没有配置 cache 策略
})
```

**影响：** 每次切换页面都重新请求数据，性能差

**修复方案：**
```typescript
const realSupabase = createClient(supabaseUrl, supabaseAnonKey, {
  global: { fetch: customFetch },
  auth: {
    storageKey: `${appId}-auth-token`,
    storage: taroStorage,
  },
  // 添加缓存策略（需要 Supabase JS v2.38+）
  // db: { schema: 'public' },
})
```

**注意：** Supabase JS 客户端的缓存需要手动管理，建议使用 React Query 或 SWR
