# 情绪化描述引擎重构报告 · 来电有喜

> 日期：2026-07-10
> 背景：用户截图反馈「智能生成情绪化描述」按钮产出的 3 条文案全是「与心上人相对而坐」式通用文艺腔，零商品特征、不可采纳。
> 结论：问题不在「词条不够」，而在**引擎架构本身**——它产出的是心情日记，不是商品文案。本次从架构层面重构，并修复一个仅在生产路径（`api.ts`）才暴露的兜底 bug。

---

## 一、根因诊断（为什么之前「全部修复」没解决）

前次修复（加 `ATTRIBUTE_PHRASES` 属性感知 + 补品类词条 + 修 category 传入）让引擎**能**感知商品，但叙事骨架仍是「纯意境诗」：

1. **商品名缺省 `|| '这件物事'`** → 商家没填名或传空时，三条文案全变成「这件物事如何如何」，根本不知道在说什么。
2. **场景词主导起笔** → `SCENE_MOMENT['约会']='与心上人相对而坐'` 被所有模板吞作开头，三条候选高度雷同（"每逢/想起/若是在…与心上人相对而坐"）。
3. **`genericMiddles` 30 条全是纯意境模板** → 只填 `${name}+${metaphor}+${realm}`，完全不描述产品本身是什么、什么味道、什么用途；即便命中属性短语，也只是泛泛意境诗的附庸。

> 前次属于「在心情日记里加了一行商品属性」，本次属于「把引擎改成商品文案生成器」。

---

## 二、重构方案（emotion-description.ts）

| 改动 | 说明 |
|------|------|
| **商品身份前置** | `buildOne` 优先用 `profile.bodyTemplates`（类目专属模板），无专属则走增强通用模板（强制含产品名 + ≥1 感官/功能维度） |
| **类目专属 bodyTemplates** | 11 业态各补 10~15 条「描述该业态产品」的接地气模板（非纯意境），如餐饮「一碗{name}端上桌时…」|
| **属性词典扩容** | `ATTRIBUTE_PHRASES` 从 ~30 扩到 50+ 条（口味/材质/产地/功效/人群/做法），`detectAttributes` 显式+隐式均走 alt 匹配并去重 |
| **场景起笔降权** | 场景词不再占主语位，改为从句或后置，消除候选雷同 |
| **`{attr}` 注入补句号** | 属性短语强制「完整句+句号」注入，根治"通透了第一口下去"类粘连病句 |
| **`pick` 防御** | 空数组返回 undefined，修复商家传「酸辣」等非 key 属性导致运行时崩溃 |
| **空名处理** | 商品名为空时返回"请先填写商品名称…"提示，不再生成废话 |

## 三、关键生产 bug（真机测试才暴露）

`src/db/api.ts:492` 的 `localCompile` 是**用户实际触发的生成入口**：

```ts
// ❌ 旧：把非空占位符喂给生成器，导致生成器空名分支永不触发
const name = payload.name || '这件物事'

// ✅ 新：空名直接返回结构化填写提示；非空才进生成器
const name = (payload.name || '').trim()
if (!name) {
  const hint = '请先填写商品名称，再生成情绪化描述——有了名字，文案才能写出它的模样。'
  return { emotion_title:'待填写商品名称', stage1:hint, stage2:hint, stage3:'', candidates:[hint], ... }
}
```

> **教训**：仅看源码/编译通过会漏掉这类入口兜底 bug。本次用 `esbuild --bundle --alias:@/client/supabase=stub --alias:@tarojs/taro=stub` 在 node 里**真机跑 `compileProductEmotion`** 才发现 `这件物事` 仍残留在运行时。

另有小修：护肤词条 `润燥|滋润|补水|保湿` 误带食品动词"吃完" → 改为"皮肤都跟着亮堂些"（跨品类泄漏）。

### 3. 真机实跑（compileProductEmotion → localCompile 兜底）
- **真实商品名**（招牌酸菜鱼 / 辣 / 约会场景）：
  > 招牌酸菜鱼端上桌时热气还冒着，海味自带一股子鲜，不用多余的佐料。第一口下去整个人就松了——像灶上咕嘟的汤，妥帖安然得刚刚好。两个人坐着慢慢吃，比什么都强。趁热，慢慢吃。
  > → 含**商品名 + 真实属性（鲜）+ 场景（两个人坐着慢慢吃）+ 品类比喻**，可直接采纳。
- **空名边界**：返回「请先填写商品名称…」提示，**无废话**。

## 六、遗留 / 后续可优化（非阻断）
