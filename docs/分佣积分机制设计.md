## 一、核心模型

### 1.1 资金流向

```
消费者支付订单
    ↓
商家收款（扣除让利部分）
    ↓
商家让利 → 平台统一收取
    ↓
平台分配：积分 + 二级分佣
```

### 1.2 三方角色

| 角色 | 定义 | 收益来源 |
|------|------|---------|
| **商家** | 提供商品和服务的店铺 | 订单收入（扣除让利后） |
| **推广员** | 平台会员（含商家员工），通过推广码带来客户 | 推广佣金（一级/二级） |
| **平台** | 来电有喜运营方 | 商家让利中的平台抽成部分 |

## 二、商家让利设置

### 2.1 让利率配置

- 商家在「店铺设置」中自行设置让利率
- 范围：**5% ~ 30%**（平台限制上下限）
- 默认值：**10%**
- 让利计算基数：**订单实付金额**（不含运费）

### 2.2 让利分配结构

```
订单金额：100 元，商家让利率：10%
─────────────────────────────────────
商家让利总额：10 元（进入平台）
├── 平台抽成：    10 元 × 20% = 2 元
├── 一级推广佣金：10 元 × 56% = 5.6 元
├── 二级推广佣金：10 元 × 24% = 2.4 元
└── 消费者积分：  10 元 × 0% = 0 元（积分另计，见第三节）
```

> **说明**：平台抽成从让利总额中出，不是从商家收入中额外扣。

## 三、积分机制

### 3.1 积分获取规则

| 行为 | 积分奖励 | 说明 |
|------|---------|------|
| 消费 1 元 | 1 积分 | 按实付金额，不含运费 |
| 首次绑定推广员 | 50 积分 | 一次性奖励 |
| 成功推荐新用户（新用户首单） | 100 积分 | 新用户完成首单后发放 |
| 每日签到 | 5 积分 | 连续签到加成，最高 20 积分/天 |
| 发布评价 | 20 积分 | 每单限一次 |

### 3.2 积分消耗规则

| 用途 | 抵扣比例 |
|------|---------|
| 抵扣现金 | 100 积分 = 1 元（最高抵扣订单金额 20%） |
| 兑换优惠券 | 见优惠券配置 |
| 兑换平台商品 | 见积分商城配置 |

### 3.3 积分有效期

- 积分有效期：**12 个月**（按获取时间滚动计算）
- 过期前 30 天、7 天、1 天各推送提醒一次

## 四、二级分佣机制

### 4.1 推荐关系链

```
推广员 A（一级）
    └── 推广员 B（二级，B 的 referrer_id = A.id）
            └── 消费者 C（C 的 referrer_id = B.id）

C 消费 → B 拿一级佣金 → A 拿二级佣金
```

### 4.2 分佣比例（从商家让利中出）

| 层级 | 比例 | 说明 |
|------|------|------|
| 一级推广员 | 让利总额的 56% | 直接带来客户的推广员 |
| 二级推广员 | 让利总额的 24% | 一级推广员的上级 |
| 平台抽成 | 让利总额的 20% | 平台运营收入 |

### 4.3 分佣结算规则

- 结算触发：**订单完成**（用户确认收货 / 超过自动确认期限）
- 结算周期：**每月 15 日**自动结算上月已完成的订单佣金
- 最低提现金额：**100 元**
- 提现到账：**1~3 个工作日**

### 4.4 推广员身份说明

推广员分为两类，分佣规则相同：

| 类型 | 定义 | 特殊权限 |
|------|------|---------|
| **普通推广员** | 平台会员申请成为推广员 | 无 |
| **商家员工** | 绑定到指定商家的推广员 | 可查看本店推广数据 |

> 商家员工也是平台推广员，只是 `store_staff` 表记录了其绑定的店铺。

## 五、客户锁定机制

### 5.1 锁定规则

- 客户**首次**通过某推广员的推广码注册/下单
- 系统自动建立永久锁定关系
- 锁定关系：**客户 ↔ 推广员**（永久，不随店铺变化）

### 5.2 跨店消费佣金

```
客户 C 被推广员 A 锁定（永久）
    ↓
C 在平台任意合作商家消费
    ↓
A 都能拿到一级推广佣金（从该商家的让利中出）
```

### 5.3 解锁条件

锁定关系为**永久**，仅在以下情况解除：

1. 客户主动申请解锁（需平台审核）
2. 推广员被平台封禁
3. 客户账号注销

> **不设自动解锁**：保证推广员的长期收益，激励推广员持续维护客户关系。

## 六、员工冲突解决方案

### 6.1 核心原则

> **首次扫码归属制 + 客户有选择权**

### 6.2 具体规则

| 场景 | 规则 |
|------|------|
| 多个员工同时让客户扫码 | 系统以**最早成功绑定**的记录为准，后续扫码无效 |
| 客户已被其他员工锁定 | 扫码时提示「您已有专属推广员」，可选择「申请更换」 |
| 客户申请更换推广员 | 需填写原因，原推广员可申诉，平台审核 |
| 员工离职 | 其锁定的客户**不转移**，仍归该员工（永久锁定） |
| 商家退出平台 | 该商家的员工推广员身份失效，但已锁定客户不变 |

### 6.3 防钻空子机制

- 同一手机号 = 同一客户（换微信/设备不生效）
- 实名认证：提现前需完成实名认证（姓名 + 身份证 + 银行卡）
- 设备指纹：检测异常注册行为

## 七、数据库关键字段设计

### 7.1 profiles 表（推广关系）

| 字段 | 类型 | 说明 |
|------|------|------|
| `referrer_id` | UUID | 上级推广员 ID（二级关系链） |
| `promoter_status` | TEXT | 推广员状态：none / pending / approved |
| `promoter_store_id` | UUID | 绑定的商家（员工推广员专用） |
| `points` | NUMERIC | 当前积分余额 |
| `total_commission` | NUMERIC | 累计佣金 |
| `settled_commission` | NUMERIC | 已结算佣金 |

### 7.2 stores 表（让利率设置）

| 字段 | 类型 | 说明 |
|------|------|------|
| `referral_rate` | NUMERIC(5,2) | 让利率 %，默认 10.00 |
| `platform_rate` | NUMERIC(5,2) | 平台抽成 %，默认 20.00（从让利中抽） |

### 7.3 orders 表（订单分佣记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| `referrer_id` | UUID | 一级推广员 |
| `referrer_level2_id` | UUID | 二级推广员 |
| `commission_amount` | NUMERIC(10,2) | 一级佣金金额 |
| `commission_amount_l2` | NUMERIC(10,2) | 二级佣金金额 |
| `commission_status` | TEXT | pending / settled / cancelled |
| `store_id` | UUID | 消费店铺 |

### 7.4 commissions 表（佣金明细）

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | UUID | 佣金受益人 |
| `order_id` | UUID | 关联订单 |
| `level` | INTEGER | 1（一级）/ 2（二级） |
| `amount` | NUMERIC(10,2) | 佣金金额 |
| `status` | TEXT | pending / settled / cancelled |
| `settled_at` | TIMESTAMP | 结算时间 |

### 7.5 points_logs 表（积分明细，已有）

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | UUID | 用户 ID |
| `order_id` | UUID | 关联订单（可选） |
| `points` | INTEGER | 积分变动（正=获得，负=消耗） |
| `type` | TEXT | earn / redeem / expire |
| `description` | TEXT | 变动说明 |
