# 来电有喜 · C 端用户研究执行包（从零启动 / Turnkey Kit）

> 用途：团队据此**直接执行**真人研究（招募→访谈/问卷/可用性测试→分析）。
> 边界说明：本包为可上场工具集；真实用户数据由人工按此采集。AI 不替代用户，仅提供方法学与执行件。
> 配套文档：`用户研究计划-来电有喜C端.md`（方法学依据）、`用户研究访谈与问卷脚本-来电有喜C端.md`（可上场脚本）。

## 1. 启动说明（5 分钟读这一节）

本执行包把"用户研究计划"变成**可操作的动作清单**。执行顺序：

1. **W0（本周）**：定稿脚本 → 过合规/法务（尤其"永久锁定/二级分销"披露话术）→ 申请招募预算与激励 → 打通小程序弹窗/社群投放通道。
2. **W1–W2**：招募 + 定性访谈（先定性，后定量）。
3. **W2–W3**：问卷投放 + 行为数据拉取（Supabase 导出）。
4. **W3–W4**：可用性测试 + 综合分析。
5. **W5–W6**：产出 Persona / 旅程图 / 优先级建议。

**角色分工建议**：研究负责人（设计研究/分析）、研究协调（招募/排期/激励发放）、记录员（访谈笔记/转录）、法务/合规（审阅知情同意与敏感话术）。

> 定性访谈 n=15（覆盖上表主分层）；问卷 n≈385（按人口分布设配额）；可用性测试 n=10（含 2 名无障碍用户）。

### 2.2 招募渠道与话术
- **小程序内弹窗**："我们在优化来电有喜，邀请你参与 30 分钟交流，送 20 元无门槛红包 / 200 积分。"
- **微信社群**：在消费者群/活动群发招募海报 + 报名表（腾讯文档/金数据）。
- **门店地推**：合作门店放二维码立牌，扫码报名领小礼品。
- **朋友圈/公众号**：推文 + 报名链接。
- **专项（无障碍）**：联系老年大学/视障协会/康复社群，或通过家属转介，单独预约。

### 2.3 筛选问卷（Screener，报名表前置）
1. 年龄：____ ；城市层级：一二线 / 三线及以下
2. 是否近 90 天在来电有喜有过行为（浏览/领红包/下单/推广）？ 是 / 否（否则排除）
3. 主要行为类型（多选）：购物 / 领红包 / 推广赚钱 / 纯逛
4. 近 30 天活跃度：每天 / 每周 / 每月 / 已很久没用
5. 是否内部员工/家属？ 是（排除）/ 否
6. 无障碍需求（选填）：老花/低视力 / 运动障碍 / 无
7. 可用时间段（多选）：工作日白天 / 晚上 / 周末

> 用筛选结果**回填 2.1 配额表**，每周同步进度，缺口渠道加码。

### 2.4 激励方案
- 定性访谈：40–60 元红包或等值积分（不影响中立性，统一标准）。
- 问卷：5 元无门槛红包 + 抽奖（大礼包）。
- 可用性测试：60–80 元 + 交通补贴（无障碍用户可上门）。
- 合规：激励在**知情同意签署后、访谈结束后**发放，避免诱导偏差。

### 2.5 知情同意流程（必做）
- 报名即展示《研究说明与同意书》（见第 5 节摘要）。
- 访谈/测试开始前，主持人**口头重申**：录音用途、可随时退出、数据脱敏。
- 签署（电子或纸质）后方可开始。

### 2.6 偏差防控清单
- [ ] 不只从"推广者社群"招人（会系统性高估分销价值）。
- [ ] 定性样本含"沉默大多数"（纯消费者、沉睡用户）。
- [ ] 无障碍用户专项招募，不遗漏。
- [ ] 主持人对所有回答保持中立，不引导"你觉得这个功能好不好"。
- [ ] 激励统一，不因回答内容差异。

## 7. 上线前启发式预检（专家评估 · 非用户数据 · 待验证）

> 基于代码与产品文档的**专家启发式评估（Nielsen 10 准则）**，仅作假设，需真实研究验证。不代表用户结论。

| # | 预检风险点 | 依据（代码/文档） | 假设（待验证） | 建议验证方式 |
|---|-----------|----------------|--------------|------------|
| H1 | 双支付通道认知负担 | `orders.payment_method`∈{wxpay, gold_beans} | 用户混淆"微信支付 vs 金豆"，结算犹豫 | 可用性任务④ + 访谈 |

## 8. 风险与应对
- **招募不足**：提前 2 周启动，多渠道并行，下沉市场加地推。
- **合规风险（分销/锁定）**：法务前置审阅所有相关话术与同意书；研究中如实记录用户态度供合规参考。
- **样本偏差**：严格按 2.1 配额回填，缺口渠道加码。
- **分析主观**：坚持三角验证，冲突结论单列不强行一致。
