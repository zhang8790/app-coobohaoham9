# 需求文档

## 1. 应用概述

### 1.1 应用名称
来店有喜 · 武林盟管理后台

### 1.2 应用描述
PC端网页管理后台，供平台超级管理员使用，连接Supabase数据库，提供商家审核、商品审核、提现审核、内容管理、用户管理等功能。采用武侠国潮设计风格，深色侧边栏配合亮色内容区，主色调朱砂红#C2410C。

## 2. 用户与使用场景

### 2.1 目标用户
平台超级管理员（role='admin'）

### 2.2 核心使用场景
- 管理员审核商家入驻申请
- 管理员审核商品上架申请
- 管理员审核用户提现申请
- 管理员管理平台UGC内容
- 管理员管理平台用户信息
- 管理员查看平台运营数据

## 3. 页面结构与功能说明

### 3.1 页面结构树

```
来店有喜 · 武林盟管理后台
├── 登录页
└── 主框架（登录后）
    ├── 左侧导航栏
    ├── 顶部Header
    └── 右侧内容区
        ├── 仪表盘（首页）
        ├── 门派大典（商家审核）
        ├── 宝贝审阅（商品审核）
        ├── 银票兑付（提现审核）
        ├── 武林贴管理（UGC内容）
        └── 用户管理
```

### 3.2 登录页

**核心功能**：管理员身份验证

**功能模块**：
- 邮箱输入框：输入管理员邮箱
- 密码输入框：输入登录密码
- 登录按钮：提交登录请求
- 权限校验：登录成功后校验用户profile.role是否为admin，非admin用户拒绝访问并提示「无权限」

### 3.3 主框架布局

#### 3.3.1 左侧导航栏

**核心功能**：功能模块导航

**功能模块**：
- 仪表盘入口：显示图标+名称，点击进入仪表盘页面
- 门派大典入口：显示图标+名称，点击进入商家审核页面
- 宝贝审阅入口：显示图标+名称，点击进入商品审核页面
- 银票兑付入口：显示图标+名称，点击进入提现审核页面
- 武林贴管理入口：显示图标+名称，点击进入内容管理页面
- 用户管理入口：显示图标+名称，点击进入用户管理页面

#### 3.3.2 顶部Header

**核心功能**：管理员信息展示与退出

**功能模块**：
- 管理员信息：展示当前登录管理员昵称或邮箱
- 退出按钮：点击退出登录，返回登录页

### 3.4 仪表盘（首页）

**核心功能**：平台关键数据展示

**功能模块**：
- 数据卡片区：展示6个数据卡片
  + 待审商家数：展示merchant_applications表中status='pending'的记录数量
  + 待审商品数：展示products表中review_status='pending_review'的记录数量
  + 待审提现数：展示withdrawals表中status='pending_audit'的记录数量
  + 文章总数：展示articles表的总记录数
  + 用户总数：展示profiles表的总记录数
  + 订单总数：展示orders表的总记录数
- 最新申请列表预览：展示最近提交的商家申请、商品审核、提现申请，每类显示最新5条，点击可跳转至对应审核页面

### 3.5 门派大典（商家审核）

**核心功能**：商家入驻申请审核

**功能模块**：
- 状态筛选：支持按状态筛选（全部/待审/已通过/已驳回）
- 商家申请列表：分页展示merchant_applications表记录，每条记录包含商家名称、申请时间、联系方式、申请状态
- 通过按钮：点击后将该申请status更新为'approved'
- 驳回按钮：点击后弹出驳回理由输入框，填写理由后将status更新为'rejected'，并保存驳回理由
- 分页控件：支持翻页查看更多申请

### 3.6 宝贝审阅（商品审核）

**核心功能**：商品上架审核

**功能模块**：
- 商品列表：分页展示products表中review_status='pending_review'的商品，每条记录包含商品图片、名称、价格、所属门店、提交时间
- 批准上架按钮：点击后将该商品review_status更新为'approved'，商品状态变更为已上架
- 驳回按钮：点击后弹出驳回理由输入框，填写理由后将review_status更新为'rejected'，并保存驳回理由
- 分页控件：支持翻页查看更多商品

### 3.7 银票兑付（提现审核）

**核心功能**：用户提现申请审核

**功能模块**：
- 提现申请列表：分页展示withdrawals表中status='pending_audit'的提现申请，每条记录包含用户昵称、用户余额、提现金额、提现方式、申请时间
- 通过按钮：点击后将该申请status更新为'approved'，执行代付操作
- 驳回按钮：点击后将该申请status更新为'rejected'
- 分页控件：支持翻页查看更多申请

### 3.8 武林贴管理（UGC内容）

**核心功能**：平台内容管理

**功能模块**：
- 发布状态筛选：支持按发布状态筛选（全部/已上架/已下架）
- 内容列表：分页展示articles表所有记录，每条记录包含文章标题、作者昵称、发布时间、发布状态
- 上架按钮：点击后将该文章状态更新为已上架
- 下架按钮：点击后将该文章状态更新为已下架
- 删除按钮：点击后删除该文章记录
- 分页控件：支持翻页查看更多内容

### 3.9 用户管理

**核心功能**：平台用户信息管理

**功能模块**：
- 用户列表：分页展示profiles表所有记录，每条记录包含用户昵称、手机号、会员段位、角色（role）、注册时间
- 修改角色：点击角色字段可修改用户role值（user/admin）
- 分页控件：支持翻页查看更多用户

## 4. 业务规则与逻辑

### 4.1 登录与权限校验
- 管理员使用邮箱+密码登录
- 登录成功后校验profile.role是否为admin
- 非admin用户拒绝访问，提示「无权限」并返回登录页
- 所有主框架页面需校验登录状态和admin权限

### 4.2 数据库连接
- 后台连接与小程序相同的Supabase数据库
- 所有数据操作直接读写Supabase对应表

### 4.3 审核流程
- 商家申请审核：通过后status变更为approved，驳回后status变更为rejected并保存驳回理由
- 商品审核：批准上架后review_status变更为approved，驳回后review_status变更为rejected并保存驳回理由
- 提现审核：通过后status变更为approved并执行代付，驳回后status变更为rejected

### 4.4 分页规则
- 所有列表页面支持分页展示
- 每页显示数量可设置为10条、20条或50条

### 4.5 数据统计规则
- 仪表盘数据卡片实时统计对应表的记录数量
- 待审数量仅统计特定状态的记录

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 非admin用户登录 | 提示「无权限」，拒绝访问 |
| 未登录访问主框架页面 | 跳转至登录页 |
| 审核操作失败 | 提示操作失败，请稍后重试 |
| 网络异常 | 提示网络异常，请检查网络连接 |
| 数据库连接失败 | 提示数据库连接失败，请联系技术支持 |
| 列表数据为空 | 展示空状态提示 |
| 分页超出范围 | 自动跳转至最后一页 |
| 修改用户角色失败 | 提示修改失败，请稍后重试 |

## 6. 验收标准

1. 管理员在登录页输入邮箱和密码，点击登录，系统校验role为admin后进入仪表盘页面
2. 管理员在仪表盘查看待审商家数、待审商品数、待审提现数等关键数据卡片
3. 管理员点击左侧导航栏「门派大典」，进入商家审核页面，查看待审商家申请列表
4. 管理员点击某条商家申请的「通过」按钮，该申请状态变更为已通过
5. 管理员点击左侧导航栏「宝贝审阅」，进入商品审核页面，对某商品点击「批准上架」，该商品状态变更为已上架
6. 管理员点击左侧导航栏「银票兑付」，进入提现审核页面，对某提现申请点击「通过」，该申请状态变更为已通过
7. 管理员点击左侧导航栏「用户管理」，进入用户管理页面，修改某用户的role字段为admin
8. 管理员点击顶部Header的「退出」按钮，退出登录并返回登录页
