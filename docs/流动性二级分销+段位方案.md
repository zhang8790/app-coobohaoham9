# 流动性二级分销 + 用户段位 完整方案

> 基于现有 V4 六段位系统，新增「流动性一级」机制
> 
> **核心架构**：流动一级（按单动态） + 静态二级（永久绑定） + 段位影响比例

---

## 一、核心概念

### 1.1 双轨分销定义

| 轨 | 定义 | 绑定方式 | 佣金计算 |
|---|---|---|---|
| **流动一级** | 本次订单的实际服务/推广人 | 按单绑定（谁服务谁拿） | 段位影响比例 |
| **静态二级** | 推荐关系链上的上级（永久） | 注册时绑定（永久） | 段位影响比例 |

### 1.2 用户段位（V4 六段位，保留不变）

动态分数 = 个人累计消费 × 30% + 团队业绩 × 70%

| 段位 | 动态分数阈值 | Icon | 颜色 |

| 外门弟子 | 500 | 🌟 | #50C878 |

## 二、分佣分配规则

### 2.1 让利池分配比例（段位影响版）

**分配公式**：
```
让利池 = 订单金额 × 商家让利率

平台最低抽成 = 让利池 × 5%（平台保底收入）
剩余池 = 让利池 - 平台最低抽成

流动一级佣金 = 剩余池 × 流动一级比例（基于服务员工段位）
静态二级佣金 = 剩余池 × 静态二级比例（基于推荐人段位）
买家积分     = 剩余池 × 积分比例（基于买家段位）
平台额外收入 = 剩余池 - 流动一级佣金 - 静态二级佣金 - 买家积分
平台总收入   = 平台最低抽成 + 平台额外收入
```

### 2.2 段位系数表

| 段位 | 流动一级系数 | 静态二级系数 | 积分系数 | 合计 | 平台收入 |
|---|---|---|---|---|---|
| 江湖散修 | 0.40 | 0.15 | 0.10 | 65% | 35% ✅ |
| 外门弟子 | 0.45 | 0.18 | 0.12 | 75% | 25% ✅ |
| 内门弟子 | 0.50 | 0.20 | 0.13 | 83% | 17% ✅ |
| 核心弟子 | 0.54 | 0.22 | 0.14 | 90% | 10% ✅ |
| 长老 | 0.57 | 0.24 | 0.15 | 96% | 4% ⚠️ |
| 掌门 | 0.60 | 0.25 | 0.15 | 100% | 0% ❌ |

> **⚠️ 平台最低抽成规则**：
> 让利池先抽 **5%** 给平台（最低保障），剩余 **95%** 再按段位系数分配。
> 这样即使掌门段位，平台也能保证 5% 收入。

### 2.3 比例上限控制（防止平台亏损）

**规则**：流动一级 + 静态二级 + 积分 ≤ 90%，剩余 ≥ 10% 归平台

```
计算步骤：
1. 按段位系数计算原始比例
2. 若总和 > 90%，等比例缩放至 90%
3. 平台最低保留 10%
```

**示例**：
- 服务员工「掌门」+ 推荐人「掌门」+ 买家「掌门」
  - 原始：75% + 30% + 20% = 125% ❌
  - 缩放后：54% + 22% + 14% = 90% ✅（平台 10%）

## 三、流动一级绑定规则

### 3.1 绑定触发条件

| 场景 | 流动一级绑定对象 |
|---|---|
| 用户扫员工推广码后下单 | 该员工 |
| 用户在小程序内直接下单（无推广码） | 最近一次服务的员工（如有） |
| 用户首次进店，无服务记录 | 不绑定流动一级（仅静态二级生效） |

### 3.2 绑定有效期

- **流动一级按单计算**，不永久绑定
- 每笔订单独立判定服务员工
- 员工可通过推广码、服务打卡等方式成为该单的流动一级

### 3.3 冲突解决（员工抢客户）

**规则：首次扫码优先 + 30天冷却**

```
用户C 第一次扫了员工A的码 → 绑定A为流动一级（有效期30天）
  ↓
30天内，C的每笔订单流动一级都给A
  ↓
30天后，若C未有新扫码行为，流动一级自动解除
  ↓
员工B可让C扫自己的码，重新绑定B为流动一级
```

**防钻空子**：
- 用户删小程序重装 → 手机号识别，绑定关系保留
- 一用户多手机号 → 实名认证（微信实名）关联

## 四、静态二级绑定规则

### 4.1 绑定触发条件

```
用户C 通过推广员A的链接/码注册
  ↓
C 的 referrer_id = A.id（永久写入 profiles 表）
  ↓
C 消费时，A 永久拿静态二级佣金
  ↓
C 推荐了D → D 的 referrer_id = C.id
  ↓
D 消费时，C 拿一级佣金，A 拿二级佣金
```

### 4.2 二级佣金衰减（防躺平）

**规则**：推荐人连续90天无新推荐，二级佣金比例每月衰减10%

```
推荐人A 最后推荐时间：2024-01-01
  ↓
2024-04-01（90天后），开始衰减
  ↓
2024-04：二级比例 = 原比例 × 90%
2024-05：二级比例 = 原比例 × 80%
...
最低衰减至原比例的 40%（封顶衰减）
  ↓
A 重新推荐新用户 → 恢复原比例
```

## 五、积分机制（段位影响）

### 5.1 积分获取

| 行为 | 积分计算 | 说明 |
|---|---|---|
| 消费获得积分 | 让利池 × 积分比例（基于买家段位） | 段位越高，积分越多 |
| 推荐新用户首单 | 100 积分（固定） | 一次性奖励 |
| 每日签到 | 5 积分（连续7天加成至20） | 平台运营池出 |
| 发布有效评价 | 20 积分 | 平台运营池出 |

### 5.2 积分消耗

| 用途 | 规则 |
|---|---|
| 抵扣现金 | 100 积分 = 1 元，最高抵扣订单金额 20% |
| 兑换优惠券 | 积分商城配置 |
| 兑换实物商品 | 积分商城配置 |

### 5.3 积分结算规则（重要！）

**用户用积分抵扣时，商家仍收全额，平台补差价**

```
订单 100 元，用户用 1000 积分抵扣 10 元
  ↓
商家到账：100 元（全额）
平台支出：10 元（积分抵扣部分）
  → 平台从运营池出这笔钱，不影响商家收入
```

## 八、分佣计算示例

### 示例 A：完整流程

**场景**：
- 订单金额：200 元
- 商家让利率：10%
- 让利池：20 元
- 服务员工：李四（段位：内门弟子，系数 0.60）
- 静态二级推荐人：张三（段位：核心弟子，系数 0.26）
- 买家：王五（段位：外门弟子，积分系数 0.12）

**计算**：
```
流动一级佣金 = 20 × 60% = 12 元 → 李四
静态二级佣金 = 20 × 26% = 5.2 元 → 张三
买家积分     = 20 × 12% = 2.4 元 → 王五（240积分）
平台收入     = 20 - 12 - 5.2 - 2.4 = 0.4 元
```

### 示例 B：段位较高，比例上限触发

**场景**：
- 让利池：20 元
- 服务员工：掌门（0.75）
- 推荐人：掌门（0.30）
- 买家：掌门（0.20）
- 原始总和：75% + 30% + 20% = 125%

**缩放后**：
```
总比例缩放至 90%：
流动一级佣金 = 20 × (75%/125% × 90%) = 20 × 54% = 10.8 元
静态二级佣金 = 20 × (30%/125% × 90%) = 20 × 21.6% = 4.32 元
买家积分     = 20 × (20%/125% × 90%) = 20 × 14.4% = 2.88 元
平台收入     = 20 × 10% = 2 元
```

## 九、防刷单/作弊风控方案

### 9.1 检测类型

| 风险类型 | 检测规则 | 风险等级 | 处理方式 |
|---|---|---|---|
| 自买自卖 | buyer_id = staff_id | 高 | 冻结佣金 |
| 闭环推荐 | 推荐关系链形成闭环（A→...→A） | 高 | 冻结佣金 |
| 高频订单 | 同一买家每日订单 > 10 笔 | 中 | 标记风险 |
| 低额订单 | 单笔 < 5 元 | 低 | 不计佣金 |
| 积分套利 | 下单拿积分 → 退款 | 中 | 追回积分 |

### 9.2 风控流程

```
订单创建
  ↓
综合风控检测（runOrderRiskCheck）
  ↓
通过 → 正常计算佣金
不通过 → 冻结佣金 + 记录风险日志
  ↓
订单完成 → 发放佣金（若未冻结）
  ↓
退款 → 追回佣金 + 追回积分
```

### 9.3 风控代码文件

- `src/utils/risk-control.ts`：风控检测函数
- 订单创建时调用 `runOrderRiskCheck()`
- 退款时调用 `recoverCommission()` + `recoverPoints()`
