# 扫码即锁客功能 - 部署指南

## 📊 功能说明

**目标**：用户扫码/点击分享链接后，即使用户未注册，也能锁定下线关系。等用户后续注册时，自动绑定邀请关系。

---

## 🛠️ 实现原理

### 1. 扫码阶段（未注册）
- 用户扫描分享二维码/点击分享链接
- 解析出推广码（`inviter=LDYX001`）
- 调用 `createPendingReferral()` 创建预锁客记录
- 记录到 `pending_referrals` 表（设备ID + 推广码 + 过期时间）

### 2. 注册阶段
- 用户注册账号
- 自动调用 `convertPendingReferral()` 转化预锁客记录
- 更新 `profiles.invited_by` 字段
- 写入 `user_store_relation` 表（锁客关系）

## 📊 数据库表结构

### `pending_referrals` 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `device_id` | TEXT | 设备标识（微信 openid 或设备ID） |
| `referral_code` | TEXT | 推广码（对应 `profiles.invite_code`） |
| `store_id` | UUID | 门店ID（可选，如果是扫码进入门店） |
| `campaign_id` | UUID | 活动ID（可选，如果是活动分享） |
| `status` | TEXT | 状态：`pending` / `converted` / `expired` |
| `converted_user_id` | UUID | 转化后的用户ID（注册后填入） |
| `created_at` | TIMESTAMPTZ | 创建时间 |
| `updated_at` | TIMESTAMPTZ | 更新时间 |
| `expires_at` | TIMESTAMPTZ | 过期时间（默认30天） |

## 🎯 关键函数

### `createPendingReferral(params)`
**调用时机**：用户扫码/点击分享链接时（不需要登录）

**参数**：
- `referral_code`：推广码
- `store_id`（可选）：门店ID
- `campaign_id`（可选）：活动ID

**功能**：
- 获取设备ID（微信环境）
- 写入 `pending_referrals` 表
- 保存到本地缓存（注册时备用）

---

### `convertPendingReferral(userId)`
**调用时机**：用户注册成功后（自动调用）

**参数**：
- `userId`：新注册的用户ID

**功能**：
- 根据设备ID查找 pending 记录
- 更新 `status` = `converted`
- 写入 `user_store_relation` 表（锁客关系）
- 更新 `profiles.invited_by`

---

## ⚠️ 注意事项

1. **设备ID生成**：使用 `Taro.getSystemInfoSync()` 获取设备信息，生成唯一设备指纹
2. **过期时间**：默认 30 天，可在迁移文件中修改 `expires_at` 默认值
3. **幂等性**：`convert_pending_referral` 函数只会转化一次（如果已经转化，不会重复绑定）
4. **隐私合规**：设备ID仅用于锁客功能，不会收集其他隐私信息

## 🐛 故障排查

### 问题 1：扫码后 `pending_referrals` 表没有记录

**排查步骤**：
1. 检查控制台日志：`[handleInviterFromQuery] 已创建预锁客记录`
2. 检查网络连接（Supabase 是否可访问）
3. 检查 RLS 策略（是否允许匿名插入）

---

### 问题 2：注册后邀请关系未绑定

**排查步骤**：
1. 检查 `pending_referrals` 表：`device_id` 是否匹配
2. 检查 `expires_at` 是否过期
3. 检查控制台日志：`[Auth] 已转化预锁客记录`

---

### 问题 3：设备ID不一致

**原因**：微信小程序环境中，设备ID可能会变化（如用户清除缓存）

**解决方案**：
- 使用微信登录（获取 openid）作为设备ID
- 或者在本地缓存中保存设备ID（`Taro.setStorageSync('device_id', ...)`）

---

## 📝 后续优化建议

1. **使用微信 openid**：如果用户授权微信登录，使用 openid 作为设备ID（更稳定）
2. **添加推广详情页**：用户扫描推广码后，先进入推广详情页（展示推广人信息、奖励等），再引导注册
3. **添加锁客有效期配置**：在管理后台添加锁客有效期配置（默认30天，可修改）
4. **添加锁客统计**：在管理后台添加锁客统计报表（预锁客数、转化率等）
