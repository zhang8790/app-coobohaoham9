# 来电有喜 · C 端用户研究脚本（可上场 / Field Instruments）

> 配套：`用户研究计划-来电有喜C端.md`、`用户研究执行包-来电有喜C端.md`
> 说明：以下为**可直接使用**的定性提纲、定量问卷、可用性测试脚本。填空题 ____ 由主持人/系统填写。
> 日期：2026-07-08

## A. 定性访谈完整提纲（45–60 min）

### A0 开场（5 min）
- 欢迎与破冰："今天不是考试，没有标准答案，聊你真实的使用感受就好。"
- 重申知情同意与录音用途，确认参与者同意。
- 说明 think-aloud："操作时把心里想的说出来；不操作时也聊聊为什么这么想。"
- 背景热身（不计入分析，仅建立信任）：平时用微信多吗？常在小程序买东西吗？

### A1 回顾与场景（10 min）
- 你第一次是怎么知道/用上来电有喜的？当时想干嘛？
- 最近一次用它是做什么？最顺的一次 / 最烦的一次经历是什么？
- 你一般**什么时间、什么场景下**会打开它？（通勤/睡前/在店门口/收到红包时？）

### A2 发现与决策（10 min）— RQ1
- 你想买东西或领红包时，一般怎么找？（搜索/首页推荐/朋友分享/门店码？）
- 什么会让你愿意在它这儿而不是美团/抖音/淘宝下单？
- 看到"情绪标签""情绪勋章"这类东西，你会因此多看或买吗？为什么？

### A3 关键流程体验（15 min）— RQ2
- 支付时，你用过"微信支付"还是"金豆"？知道两者区别吗？有没有犹豫过？
- 你领过红包/礼品吗？领完之后，**去哪、怎么用掉**的？顺不顺？
- 积分和段位（江湖散修→掌门）对你继续用有多大影响？有没有"白攒了"的感觉？
- 你有没有生成过推广码 / 被别人推荐进来？知道自己和某个推广员"绑定"了吗？

### A4 价值与态度（10 min）— RQ3
- 如果用一句话形容来电有喜，你会说什么？
- 哪些功能让你觉得"这 App 懂我 / 对我有用"？哪些觉得"花哨没用"？
- 假设有个"完美版"，你最想要哪 3 个改进？

### A5 收尾（5 min）
- 还有什么没聊到但你想说的？
- 感谢 + 告知激励发放方式。

> **追问技巧**：用"为什么""能举个例子吗""当时你心里怎么想"代替引导性提问；避免"你觉得这个功能好不好"。
> **记录**：主持人负责主线，记录员同步记引语与情绪（😀😐😣）。

## B. 定量问卷量表（线上，约 8–10 min）

> 题型：单选 / 多选 / 李克特（1=非常不同意 … 5=非常同意）/ 排序。配额见执行包 2.1。

### B1 人口统计与分层（用于配额与分群）
- 年龄：18–25 / 26–35 / 36–45 / 46+
- 城市层级：一二线 / 三线及以下
- 近 30 天活跃度：每天 / 每周 / 每月 / 已很久没用
- 主要身份（多选）：纯消费者 / 领红包党 / 推广者 / 商家(若有)

### B2 需求优先级（多选 + 排序）
"你用来电有喜最看重什么？" 选项：省钱 / 赚钱(推广) / 便利(就近) / 情绪满足 / 社交 / 信任/安全感 / 积分权益
→ 再请排序 Top3。

### B3 痛点发生率（近 90 天是否遇到，是/否 + 频率：偶尔/经常）
- 支付时不知道选微信还是金豆
- 领了红包不知道去哪核销 / 核销失败
- 找不到想买的商品或门店
- 不理解自己被某推广员"绑定"
- 觉得积分/段位没用或规则看不懂
- 情绪/勋章功能看不懂或用不上
- 其他（填空）

### B4 态度与价值量表（1–5 李克特）— RQ3
- 我能清楚说出来电有喜对我有什么用
- 积分/段位让我更愿意回来
- 领红包/礼品是我在意的功能
- 情绪相关功能帮我发现想要的东西
- 我知道自己是否被某个推广员绑定，且对此感到安心
- 整体来说，来电有喜让我觉得"划算/值得"

### B5 满意度与忠诚
- 总体满意度（1–5）
- 推荐意愿 NPS（-100 ~ +100，0–10 量表）
- 继续使用的可能（1–5）

### B6 开放题（选填）
- 最想改掉的 1 个问题：____
- 最想加的 1 个功能：____

## C. 可用性测试全程脚本（60 min，moderated）

### C0 准备
- 设备：被测者自有手机（微信 + 来电有喜小程序，登录测试账号或本人账号）。
- 录屏 + 录音授权（见知情同意）。
- 角色：主持人（引导）、观察员（记指标）、记录员（笔记）。

### C1 介绍（5 min）
- 欢迎、think-aloud 说明、背景问题（同 A0 热身）。

### C2 基线（5 min）
- 你现在用来电有喜主要干嘛？用过哪些功能？

### C3 任务场景（35 min）
> 每任务记录：完成率(是/否)、耗时(秒)、错误数、求助次数、情绪(😀😐😣)。成功标准见每任务。

**任务① 首单下单**
- 场景："你想买一杯附近门店的奶茶，找到它、下单并用微信支付完成。"
- 成功：支付成功。观察：搜索/发现效率、配送vs自提选择、支付犹豫。

**任务② 领红包并核销**
- 场景："你领了一个门店红包/礼品，去对应门店把它用掉。"
- 成功：核销成功。观察：是否知道去哪核销、核销门槛阻碍。

**任务③ 情绪发现**
- 场景："今天有点累，用'情绪'相关功能找件让自己开心的小东西。"
- 成功：找到并进入商品。观察：是否理解情绪入口、是否真参考。

**任务④ 积分抵扣**
- 场景："结算时，用积分抵一部分钱。"
- 成功：抵扣生效。观察：规则认知、是否达上限困惑。

**任务⑤ 段位/推广认知**
- 场景："看一下你的段位进度；如果你有推广码，看看自己是否被绑定。"
- 成功：能说出当前段位/是否知晓绑定。观察：成长反馈感、锁定披露感知。

### C4 事后问卷（10 min）
- SUS 系统可用性量表（10 题，见下）。
- 任务难度评分（每任务 1–5）。
- 开放反馈："哪个任务最卡？为什么？"

### C5 SUS 量表（标准化，1=非常不同意 … 5=非常同意）
1. 我愿意频繁使用这个系统
2. 这个系统不必要的复杂
3. 我觉得用起来很容易
4. 我需要技术支持才能用
5. 各项功能整合得很好
6. 系统里有很多不一致
7. 我想象大多数人能快速上手
8. 用起来很麻烦
9. 我对使用很有信心
10. 用之前我需要学很多

> SUS 计分：奇数题 (得分-1)，偶数题 (5-得分)，求和×2.5 → 0–100。≥68 为及格。
