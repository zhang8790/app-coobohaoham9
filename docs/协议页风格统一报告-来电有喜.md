# 协议页风格统一报告 · 来电有喜

> 报告时间：2026-07-10
> 承接任务：全站合规红线审计报告（2026-07-10）遗留建议③「协议页主题 token 化 + 全局字号阶梯」
> 范围：9 个协议/规则类页面
> 结论：✅ 已全部统一为语义 token，并修复浅色模式下文字不可读的真实 bug

## 一、背景与诊断

上次全站合规审计的风格检查项指出：协议页存在「硬编码色值 / 字号散落 / 字形混用」三类问题（非红线，建议后续优化）。本次「继续」即落地下此任务。

**关键诊断（比"夜间模式失效"更紧迫的真实 bug）**：
- 协议页文本色用 `#E5E7EB`（近白）、`#9CA3AF`（浅灰）、`#6B7280`（中灰）、`#1F2937`（深灰边框）。
- 这套配色明显为**深色背景**设计，但本项目卡片容器是 `bg-card`（CSS 变量 `--card: 0 0% 100%` = 纯白）。
- 后果（浅色模式，即当前唯一主题）：
  - 标题 `#E5E7EB` 在白底上对比度 **~1.1:1** → 几乎隐形，用户根本读不到标题。
  - 正文 `#9CA3AF` 对比度 **~2.8:1** → 低于 WCAG AA 要求的 4.5:1，可读性差。
  - 边框 `#1F2937`（深）在白底上可见，但语义应为浅灰边框。

**额外发现（合规残留）**：
- `merchant-agreement` 第 22 行仍有「推广分佣（L1/L2）」，与「二级推广，只有二级」口径不一致。
- `rank-rules` 段位对照表表头 `L1/L2` 及说明文字，同为残留。

## 二、统一规范（Token 映射表）

| 元素 | 原写法（硬编码） | 新写法（语义 token） | 目的 |
|------|----------------|---------------------|------|
| 主标题 | `color:#E5E7EB; fontSize:16; fontWeight:700` | `text-foreground text-base font-bold` | 近黑高对比，替代不可见的近白 |
| 小节标题 | `color:#E5E7EB; fontWeight:600` | `text-foreground font-semibold` | 近黑，区分于正文 |
| 正文段落 | `color:#9CA3AF; fontSize:13` | `text-muted-foreground text-sm leading-loose` | 中深灰 5.7:1，达 WCAG AA |
| 次要/日期/底部 | `color:#6B7280; fontSize:11/12` | `text-muted-foreground text-xs` | 统一次要层级 |
| 分隔边框 | `borderTop:'1px solid #1F2937'` | `style={{ borderTop:'1px solid hsl(var(--border))' }}` | 语义边框变量 |
| 字形 | `fontWeight:700 / 600 / 'bold'` | `font-bold / font-semibold` | 消除三种写法混用 |

> 语义 token 来自 `tailwind.config.js` 的 `colors` 映射（基于 `app.scss` 的 HSL CSS 变量），`commission-rules` 本身已在用 `bg-card`/`border-border` 类，证明该机制在构建期有效。

## 三、涉及文件与改动

### 纯文本协议页（6 个，同构处理）
- `commission-rules`（推广规则）
- `distribution-agreement`（推广服务协议）：风险警示区 `#FBBF24` → `text-amber-600`、`#E5E7EB` → `text-foreground`，保留 amber 浅底背景
- `merchant-agreement`（商家入驻协议）：**[顺带修复合规残留]** 「推广分佣（L1/L2）」→「推广分佣（直推+间推）」
- `points-rules`（积分规则）
- `privacy-policy`（隐私政策）
- `trade-rules`（交易规则）
- `user-agreement`（用户服务协议）
- `withdraw-rules`（提现规则）

### 含表格页（1 个，特殊处理）
- `rank-rules`（段位规则）：
  - 段位对照表原为深色块（`#1F2937` 深底 + `#E5E7EB` 浅字 + `#111827` 行底），**整表改为浅色斑马纹**（偶数行 `hsl(var(--muted))` 浅灰、奇数行透明），边框 `hsl(var(--border))`。
  - 表头 `L1/L2` → **`直推/间推`**；表内文字 `#D1D5DB` → `text-muted-foreground`。
  - 段位名保留各色 `r.color`（金/银/铜/蓝/绿为品牌强调色，非灰度文本，合理保留）。
  - 底部说明「L1/L2 为佣金比例」→「直推/间推为佣金比例」。

## 五、注意事项与后续

1. **当前主题状态**：`app.scss` 仅定义了 `:root` 浅色变量，**无 `.dark` 覆盖**，`tailwind.config.js` 也未配 `darkMode`。即全站目前只有浅色主题。本次 token 化已让协议页在浅色模式下可读，并为未来接入 dark mode 自动适配铺好路（token 会自动随主题翻转）。
2. **rank-rules 段位强调色**（`#D4AF37` 金 / `#CD7F32` 铜 / `#4A90D9` 蓝 / `#50C878` 绿 / `#90EE90` 浅绿 / `#9CA3AF` 银）属于段位徽章品牌色，非灰度文本，按设计保留。
3. **范围边界**：本次仅统一协议/规则类页面。全站其它业务页仍存在内联 `fontSize` 9px~48px 散落，未做全局字号阶梯抽取。若需全站字号统一，可另立项（低优先、非红线）。
