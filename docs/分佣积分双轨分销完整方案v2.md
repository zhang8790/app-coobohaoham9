> 版本：v2.0（整合版）  
> 日期：2026-07-03  
> 状态：待最终确认

## 一、核心模型

### 1.1 资金流向（总览）

```
消费者支付订单（实付金额）
    ↓
商家收款 = 订单金额 - 商家让利
    ↓
商家让利（商家自设 5%~30%）→ 平台统一收取
    ↓
平台分配（从让利总额中出，不额外扣商家）：
    ├── 一级推广佣金（流动，按单计算）
    ├── 二级推广佣金（静态，按推荐链永久计算）
    ├── 消费者积分
    ├── 平台运营池（20%，用于补贴、运营、风控）
    └── 商家补贴（从运营池返还，见第七节）
```

### 1.2 双轨架构说明

| 轨道 | 性质 | 计算基准 | 归属 |
|------|------|---------|------|
| **流动一级** | 动态 | 按单计算，谁服务谁拿 | 本单服务推广员 |
| **静态二级** | 永久 | 按推荐链计算，永久有效 | 推荐链上级推广员 |

> **核心区别**：流动一级随订单流动，静态二级永久绑定推荐关系，两条轨道独立计算、独立发放。

## 二、商家让利设置

### 2.1 让利率配置

- 商家在「店铺设置」中自行设置让利率
- 范围：**5% ~ 30%**（平台限制上下限）
- 默认值：**10%**
- 让利计算基数：**订单实付金额**（不含运费、不含积分抵扣部分）

### 2.2 让利分配比例（默认）

```
订单金额：100 元，商家让利率：10%，行业：餐饮（高频）
────────────────────────────────────────
商家让利总额：10 元（进入平台统一分配）
├── 流动一级佣金：10 元 × 56% = 5.6 元（给本单服务推广员）
├── 静态二级佣金：10 元 × 24% = 2.4 元（给推荐链上级）
├── 平台运营池：  10 元 × 20% = 2.0 元（平台支配，用于补贴/风控）
└── 消费者积分：  由平台运营池另行发放（见第三节，不从让利扣）
```

> **说明**：平台运营池（20%）是平台收入，同时承担所有新增补贴成本，不增加商家负担。

## 三、积分机制

### 3.1 积分定位与资金来源

积分是平台用于**提升用户复购、增强用户粘性**的运营工具，**不从商家让利中扣除**，全部由平台运营池（让利总额的 20%）承担成本。

积分 ≠ 现金，不可直接提现，只能在本平台生态内消耗。

### 3.2 积分获取规则

| 行为 | 积分奖励 | 发放触发条件 |
|------|---------|---------|
| 消费 1 元 | 1 积分 | 订单完成（用户确认收货） |
| 首次绑定推广员 | 50 积分 | 绑定成功即时发放 |
| 成功推荐新用户首单 | 100 积分 | 被推荐人完成首单，订单完成 |
| 每日签到 | 5 积分（连续 7 天以上 10 积分，14 天以上 15 积分，30 天以上 20 积分） | 每日首次打开小程序 |
| 发布有效评价 | 20 积分 | 评价审核通过（不含辱骂/无用内容） |
| 平台活动奖励 | 按活动规则 | 活动页参与并完成指定任务 |

> **说明**：积分发放由平台承担成本，商家无需出钱，但积分抵扣消费时，商家收到的是等值现金（平台与商家结算时补足积分部分）。

### 3.3 积分抵扣结算规则（关键）

```
用户 C 下单 100 元，使用 500 积分抵扣 5 元
    ↓
C 实际支付：95 元
商家收款：100 元（全额）
    ↓
平台在结算给商家时，额外向商家支付 5 元（积分抵扣部分）
    ↓
平台成本：5 元（从运营池出）
```

> **核心**：积分抵扣不减少商家收入，平台承担积分成本，商家愿意支持积分支付。

### 3.4 积分消耗规则

| 用途 | 规则 |
|------|------|
| 抵扣现金 | 100 积分 = 1 元，最高抵扣订单金额 20% |
| 兑换优惠券 | 消耗积分，生成优惠券，无现金成本 |
| 兑换平台商品 | 积分商城实物/虚拟商品，平台采购成本从运营池出 |
| 捐赠公益 | 可选功能，积分直接销毁，不发放实物 |

### 3.5 积分有效期与提醒

- 有效期：**12 个月**（按单笔获取时间滚动计算，非统一到期）
- 过期前提醒：**30 天、7 天、1 天**各推送一次
- 过期后积分自动销毁，不可恢复
- 用户可在「我的积分」页面查看每笔积分的到期时间

### 3.6 积分与分佣的关系

| 项目 | 积分 | 分佣 |
|------|------|------|
| 受益人 | 消费者（下单人） | 推广员（带来客户的人） |
| 资金来源 | 平台运营池 | 商家让利 |
| 是否现金 | 否（仅平台内使用） | 是（可提现） |
| 与订单关系 | 按消费金额获得 | 按让利金额计算 |
| 跨店是否有效 | 是（全平台通用） | 是（静态二级永久，流动一级按单） |

> **总结**：积分是给消费者的「返利」，分佣是给推广员的「佣金」，两者独立，资金来自不同池子。

## 四、双轨分佣机制

### 4.1 推荐关系链（静态二级）

```
推广员 A（顶级，无上级）
    └── 推广员 B（B.referrer_id = A.id）
            └── 消费者 C（C.referrer_id = B.id）

C 消费 → B 拿静态二级佣金（一级） → A 拿静态二级佣金（二级）
```

> 静态二级关系**永久有效**，不受客户跨店消费影响。

### 4.2 流动一级（动态）

```
消费者 C 到店消费
    ↓
C 本次是通过哪个推广员的码/链接来的
    ↓
本单流动一级佣金给该推广员
    ↓
与静态二级关系无关，按单独立计算
```

> **关键**：同一客户同一笔订单，流动一级和静态二级**可以同时发放**（两个推广员各拿各的）。

### 4.3 分佣比例（从商家让利中出）

| 项目 | 比例 | 说明 |
|------|------|------|
| 流动一级佣金 | 让利总额的 56% | 本单服务推广员 |
| 静态二级佣金（一级） | 让利总额的 24% | 推荐链直接上级 |
| 静态二级佣金（二级） | 从 24% 中按平台规则分配 | 推荐链间接上级 |
| 平台运营池 | 让利总额的 20% | 平台收入 + 补贴资金池 |

### 4.4 结算规则

- 结算触发：**订单完成**（用户确认收货 / 超过自动确认期限）
- 结算周期：**每月 15 日**自动结算上月已完成的订单佣金
- 最低提现金额：**100 元**
- 提现到账：**1~3 个工作日**
- 流动一级和静态二级**分开结算**、**分开提现**

## 五、客户锁定与流动规则

### 5.1 静态二级锁定（永久）

- 客户**首次**通过某推广员的推广码注册/下单
- 系统自动建立静态二级关系（`profiles.referrer_id`）
- **永久有效**，不随消费行为变化

### 5.2 流动一级（按单计算，不锁定）

- 流动一级**不锁定客户**，每笔订单独立判断
- 判断依据：客户下单时使用的是哪个推广员的码/链接
- 无推广码下单：本单无流动一级佣金，仅发放静态二级佣金

### 5.3 跨店消费规则

```
客户 C 的静态二级 = 推广员 B（永久）
    ↓
C 在商家 X 消费 → 静态二级佣金给 B（不受影响）
                → 流动一级佣金给 X 店的服务推广员
    ↓
C 在商家 Y 消费 → 静态二级佣金仍给 B
                → 流动一级佣金给 Y 店的服务推广员
```

> 跨店消费不影响静态二级收益，但流动一级随订单流转。

## 六、员工端补贴与冲突解决方案

### 6.1 同店长期复购平台专项补贴

**问题**：不会裂变的纯接待店员无长期被动收入，收入差距大。

**方案**：
```
同一客户 30 天内在本店累计 3 笔及以上有效订单
    ↓
每多一笔，给本店成交员工发放复购补贴
    ↓
补贴金额 = 该笔订单让利金额 × 30%（上限 50 元/笔）
    ↓
资金来自平台运营池，不占用商家让利
```

### 6.2 熟客回流激励补贴

**问题**：熟客偶然跨店，本店直接丢失持续流动一级收益。

**方案**：
```
客户跨店消费后，90 天内再次回到本店成交
    ↓
平台一次性补发回流补贴给本店员工
    ↓
补贴金额 = 上一笔跨店订单的流动一级佣金金额 × 80%
    ↓
每客户每 90 天限触发 1 次
```

### 6.3 门店熟客标签（权重优先）

**问题**：员工为抢流动一级收益过度竞争。

**方案**：
```
门店持续服务同一客户满 4 笔订单
    ↓
系统给该客户打上「本店熟客」标签
    ↓
该客户后续回本店消费：流动一级 100% 给本店员工
该客户去其他店消费：流动一级仍按正常规则（不强行分配）
```

### 6.4 二级收益平滑保底

**问题**：二级分销收益波动大，员工收入预期不稳定。

**方案**：
```
单月静态二级佣金 < 平台设定的基础阈值（如 200 元）
且当月该员工有至少 1 笔有效二级佣金产生
    ↓
平台运营池补足差额至阈值
    ↓
连续 3 个月无有效二级佣金 → 取消保底资格
```

### 6.5 员工离职二选一柔性方案

**问题**：离职后二级佣金 180 天到期清零，易引发劳动纠纷。

**方案**：员工离职时**自主选择**以下一种：

| 方案 | 说明 |
|------|------|
| **A：过渡期领取**（原有） | 180 天内按月领取二级佣金，到期后终止 |
| **B：一次性结清**（新增） | 按近 6 个月平均二级收益 × 1.5 倍一次性结清，结清后永久终止二级分佣 |

> 员工在离职流程中勾选，系统自动执行，减少人工仲裁。

## 七、商家端补贴与激励方案

### 7.1 分层让利梯度配套扶持

**问题**：薄利门店让利压力大，高让利门店需要更多流量。

**方案**：

| 让利率档位 | 平台抽成 | 额外扶持 |
|------------|---------|---------|
| 3%~5%（薄利） | 下调至 15% | 多出 5% 运营池全部返还门店作为经营补贴 |
| 6%~11%（标准） | 维持 20% | 无 |
| 12%~30%（引流） | 维持 20% | 额外给到门店流量曝光扶持 |

### 7.2 门店收益对冲补贴

**问题**：让利成本高的门店净支出压力大。

**方案**：
```
平台按月统计门店综合让利成本
    ↓
成本高于行业均值的门店
    ↓
次月从平台运营池发放补贴，返还超出部分的 50%
```

### 7.3 拉新溯源奖励

**问题**：门店引流拉新，客户外流给别家做嫁衣。

**方案**：
```
门店通过线下活动、地推带来的全新注册用户
    ↓
标记「源头门店」
    ↓
该用户 180 天内任意跨店消费
    ↓
流动一级佣金的 20% 溯源返还给源头门店员工
    ↓
180 天后终止溯源返还
```

### 7.4 全域活动平台全额兜底

**问题**：商家不愿单独让利做活动，担心亏损。

**方案**：
```
平台统一发起的全域引流活动
    ↓
拓客成本由平台运营池承担
    ↓
商家参与活动无需单独降价让利
    ↓
活动期间商家正常收单，让利部分由平台补贴
```

## 九、风控与作弊防范

### 9.1 双重扣分处罚机制

```
检测到可疑订单（关联账号互刷、极速退款、24 小时高频滚单）
    ↓
直接同时作废：流动一级佣金 + 静态二级佣金（全部层级）
    ↓
单次轻微套利：扣除近 15 天全部双重佣金
    ↓
大幅提升作弊成本
```

### 9.2 分层风控校验（节省算力）

| 校验类型 | 执行时机 | 内容 |
|---------|---------|------|
| 轻量实时校验 | 订单高峰时段 | 拦截关联账号互刷、极速退款 |
| 夜间离线批量校验 | 凌晨 2:00~6:00 | 流转闭环检测、固定重复流转路径分析 |

### 9.3 闭环作弊追加处罚

```
检测到小号闭环流转（A→B→C→A）
    ↓
冻结相关账号 30 天内所有佣金
    ↓
追加处罚：该推广员所有下线静态二级佣金暂停发放，复核后恢复
```

### 9.4 员工黑名单机制

```
员工累计 3 次刷单/套利记录
    ↓
列入平台推广员黑名单
    ↓
永久禁止成为平台推广员（含所有商户）
    ↓
同步至实名认证系统，阻止换号注册
```

## 十、用户体验优化

### 10.1 订单绑定临时服务入口

```
每笔成交记录：
    ├── 显示本期服务员工昵称 + 头像
    └── 客户发起售后：一键联系该员工（带聊天/电话入口）
    ↓
客户无专属固定员工，但每单都有对接人
    ↓
平台客服兜底，弥补无固定绑定的短板
```

### 10.2 拒绝推销标签

```
客户标记「拒绝推销」标签
    ↓

    ↓
有效期内：所有推广员不得主动推介项目
    ↓
违规推介：扣减该笔订单佣金 50%
    ↓
门店推销类投诉纳入门店服务分考核
```

## 十一、模式可选（商家自主切换）

### 11.1 两种模式说明

| 模式 | 适用 | 流动一级 | 静态二级 |
|------|------|---------|---------|
| **模式 A（默认）** | 餐饮、商超、高频零售 | 按单流动 | 永久有效 |
| **模式 B（可选）** | 医美、教培、健身、社区生鲜 | 阶梯锁客（12+12 个月） | 永久有效 |

### 11.2 模式 B 阶梯锁客规则

```
客户首次绑定员工/门店
    ↓
第 1~12 个月：全额分佣（流动一级 100% 给绑定员工）
第 13~24 个月：衰减分佣（流动一级按 50% 发放，剩余 50% 流入全域公域）
第 25 个月起：流入全域公域（流动一级按正常跨店规则计算）
    ↓
静态二级不受模式切换影响，永久有效
```

### 11.3 切换过渡规则

```
切换为模式 B：
    ├── 已有流动一级关系的客户 → 继续按原规则执行至自然到期
    └── 新客户 → 按模式 B 阶梯锁客

切换回模式 A：
    ├── 阶梯锁客未到期的客户 → 剩余有效期继续按阶梯发放，不强行解锁
    └── 新客户 → 按模式 A 流动计算
```

## 十二、数据库关键字段设计

### 12.1 profiles 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `referrer_id` | UUID | 静态二级上级推广员 ID（永久） |
| `promoter_status` | TEXT | none / pending / approved |
| `promoter_store_id` | UUID | 绑定的商家（员工推广员专用） |

| `total_commission` | NUMERIC | 累计佣金（含流动+静态） |
| `settled_commission` | NUMERIC | 已结算佣金 |
| `refuse_promote_tag` | TIMESTAMP | 拒绝推销标签过期时间（NULL=无） |

### 12.2 stores 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `referral_rate` | NUMERIC(5,2) | 让利率 %，默认 10.00 |
| `distrib_mode` | TEXT | 分销模式：flow（模式A）/ ladder（模式B） |
| `industry_tag` | TEXT | 行业标签：高频/低频高客单 |

### 12.3 orders 表

| 字段 | 类型 | 说明 |

## 附录：分佣计算完整示例

### 示例场景

```
推广员 A（顶级，无上级）
    └── 推广员 B（员工，绑定门店 X）
            └── 消费者 C（C.referrer_id = B.id，永久）

C 到门店 X 消费 500 元，商家让利率 10%
C 本次下单是通过推广员 B 的码
```

### 佣金计算

```
让利总额 = 500 × 10% = 50 元

流动一级佣金 = 50 × 56% = 28 元 → 给 B
静态二级佣金 = 50 × 24% = 12 元
    └── B 无上级（A 是顶级，无二级） → 12 元全部发放给 B（或归入平台运营池）

平台运营池 = 50 × 20% = 10 元

消费者积分 = 500 积分 → 从平台运营池成本支出
```

### 若 A 不是顶级（有上级 D）

```
静态二级佣金 = 12 元
    ├── 一级（B）= 12 × 70% = 8.4 元 → 给 B
    └── 二级（D）= 12 × 30% = 3.6 元 → 给 D
```
