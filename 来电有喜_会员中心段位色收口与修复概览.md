# 来电有喜 · 会员中心段位色收口与修复概览

> 本文记录「会员中心（user 页）段位配色」从硬编码主色收敛为统一主题 token、并修复段位色引用错乱的过程。
> 对应提交：`a1e4a75`（refactor(主题token): 收口品牌主色 #A8552E → hsl(var(--primary)) + 修复 user 页段位色引用）、`3f72b65`（style(主题): token 收口专项）。

## 1. 问题背景
- 品牌主色 `#A8552E` 在大量页面被**硬编码**写入，主题切换（light/dark）时无法跟随，且各页口径不一；
- `src/pages/user/index.tsx`（会员中心）的**段位色引用错乱**：段位徽标/进度条使用的颜色变量与全局段位定义不一致，出现「同段位不同色」或段位高亮失效；
- 卡片、三态（空态/页头/骨架）等基础 UI 缺少统一 token 封装，散落样式难维护。

## 2. 改造方案
### 2.1 品牌主色 token 化（a1e4a75）
- 全量将硬编码 `#A8552E` 替换为 `hsl(var(--primary))`，使主色跟随主题变量；
- 受影响页面（11 个文件）：`admin/index`、`merchant-center`、`merchant-products`、`my-promotion`、`store-home`、`trade/commission-detail`、`trade/tongbao-ledger`、`trade/withdraw`、`user` 等；
- **重点修复**：`src/pages/user/index.tsx` 的段位色引用，使其正确指向统一段位配色，段位徽标/进度与全局一致。

### 2.2 主题 token 收口专项（3f72b65）
- 卡片类样式统一收口为主题 token；
- 新增基础 UI 组件：`src/components/ui/EmptyState.tsx`、`PageHeader.tsx`、`Skeleton.tsx`，统一三态展示；
- 配合 `b935d4b` 的全页大厂标准评审，对 `food-scan`、`health-profile`、`index`、`messages`、`tongbao-ledger` 等页面做布局/主题一致性修复。

## 3. 收益
- 会员中心段位配色与全局主题一致，主题切换时正确跟随；
- 主色单点定义，换肤/改色零成本；
- 三态组件复用，减少重复样式与不一致。

## 4. 验证
- `taro build --type weapp` 通过（EXIT=0）；
- 真机核对：会员中心段位徽标、进度条配色在 light/dark 下均正确且一致。
