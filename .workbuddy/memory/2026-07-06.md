## 修复商家中心加载问题（最终版）

### 问题
商家中心一直显示"加载中"，无法进入主界面。

### 根本原因
- `useEffect` 中 `Promise.all` 同时加载商家信息 + 统计数据（商品+订单）
- 如果统计数据查询慢或挂起，整个页面被阻塞
- 没有超时保护机制

### 解决方案
**拆分加载逻辑为两步：**
1. **第一步（快速）**：加载商家信息 + 审核状态 → 立即退出 loading，显示页面
2. **第二步（异步）**：加载统计数据（商品+订单）→ 异步更新，不阻塞UI
3. **添加认证检查**：先检查用户是否登录
4. **添加超时保护**：5秒后强制退出加载状态

**代码改动：**
- `src/pages/merchant-center/index.tsx`
- 使用 `Promise.allSettled` 分别处理错误
- 添加详细错误日志
- 代码已提交（commit `a1e9cb2`）

## 修复侠客（个人中心）页面加载中问题 (commit 33a53c5)

### 问题
侠客页面的"商家申请入口"区域一直显示"加载中..."

### 根因
`loadData` 中 `Promise.all([getMyProfile(), getMyMerchantApplication(), getOrderCounts()])` 任一API请求失败会导致整体reject，`profile` 永远为 null → 第98行判断 `!profile` 为 true → 始终显示"加载中..."

### 修复
给每个请求添加独立的 `.catch()` 错误处理，任一请求失败不影响其他数据加载。
