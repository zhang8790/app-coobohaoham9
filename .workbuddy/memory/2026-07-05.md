## 扫码即锁客功能（本日晚间）

### 需求
用户扫码/点击分享链接后，即使用户未注册，也能锁定下线关系。等用户后续注册时，自动绑定邀请关系。

### 实现方案

**核心原理**：
1. 扫码阶段（未注册）→ 创建 `pending_referrals` 记录（设备ID + 推广码）
2. 注册阶段 → 转化 `pending_referrals` 记录（绑定邀请关系）

**文件变更**：
- 新增 `supabase/migrations/00035_add_pending_referrals.sql`（创建表和函数）
- 修改 `src/db/api.ts`（新增 `createPendingReferral`、`convertPendingReferral` 函数）
- 修改 `src/utils/share.ts`（扫码时创建预锁客记录）
- 修改 `src/contexts/AuthContext.tsx`（注册成功后转化预锁客记录）

### 数据库变更

**新增表**：`pending_referrals`（预锁客记录）
| 字段 | 说明 |
|------|------|
| `device_id` | 设备标识（微信 openid 或设备ID） |
| `referral_code` | 推广码（对应 `profiles.invite_code`） |
| `store_id` | 门店ID（可选） |
| `campaign_id` | 活动ID（可选） |
| `status` | 状态：`pending` / `converted` / `expired` |
| `converted_user_id` | 转化后的用户ID |
| `expires_at` | 过期时间（默认30天） |

**新增函数**：
- `convert_pending_referral(p_device_id, p_user_id)` — 转化预锁客记录
- `update_expired_pending_referrals()` — 自动过期

### API 函数

| 函数 | 调用时机 | 功能 |
|------|----------|------|
| `createPendingReferral` | 扫码/点击分享链接时 | 创建预锁客记录 |
| `convertPendingReferral` | 注册成功后 | 转化预锁客记录 |
| `checkAndBindReferralCode` | 注册成功后（备用） | 绑定本地缓存的推广码 |

### 测试方法

**测试场景 1**：扫码不注册，后续注册
1. 扫描分享二维码（带 `inviter=LDYX001` 参数）
2. **不要注册**，直接关闭小程序
3. 重新打开小程序，注册新账号
4. 查看数据库：`pending_referrals` 表应该有记录（`status` = `converted`）

**测试场景 2**：扫码后立即注册
1. 扫描分享二维码
2. 立即注册账号
3. 查看数据库：邀请关系应该已绑定
