## Mock层 select() 覆盖 bug 修复 + 商品详情"不存在"排查
- **根因**: `select()` 方法会把 `_action` 改回 `'select'`，导致 `.insert(data).select().maybeSingle()` 链式调用时 insert 动作被覆盖
  - insert → _action='insert'
  - select → _action='select' 💥 被覆盖！
  - execute() → 执行 SELECT 而非 INSERT
- **修复**: `select()` 不再覆盖 _action，只设置 _columns
- **INSERT 支持**: 数组形式 insert([a,b]) + 统一写入 store[table]
- **UPDATE 修复**: update 后接 .select() 正确返回更新后数据
- **createProduct 增强**: 新建商品前先查门店信息，返回数据补上 stores 关联
- 编译结果: 14.16s, 零错误
- 用户反馈: 商品详情页显示"商品不存在"，需清除缓存重新编译验证

## V4终极分佣算法（防躺平版）- 完整实现
- **核心问题**: 上级躺平不消费，只靠下级拿佣金，平台亏损
- **三大机制**:
  1. **个人活跃门槛**: 分佣资格开关
     - 连续2个月零消费 → 取消分佣资格
     - 当月零消费（首次）→ 一级佣金减半
     - 当月消费≥39元 → 全额分佣
  2. **团队流水阶梯**: 动态佣金池
     - 低档（GMV<1000）→ 平台抽10%，压缩佣金支出
     - 中档（1000≤GMV<5000）→ 平台抽8%（标准）
     - 高档（GMV≥5000）→ 平台抽7%，让利团队
  3. **拓新衰减机制**: 只奖励持续拓新
     - 有新增下线 → 恢复全额佣金（L1权重65%，L2权重35%）
