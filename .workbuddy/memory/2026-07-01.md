## 全页面空白问题修复（根因：withRouteGuard HOC 编译丢失）

### 问题现象
- 微信开发者工具中所有页面显示空白（导航栏和 TabBar 正常，内容区域全白）
- 首页、探索、犒赏铺、行囊、侠客——全部 Tab 页 + 子页面均空白

### 根因定位
**`withRouteGuard` HOC（高阶组件）模式在 Taro 4.x + Rollup 编译后函数引用丢失**
- 之前只从 `pages/index/index.tsx` 移除了 withRouteGuard
- 但 **还有 28 个页面仍在使用** `export default withRouteGuard(PageComponent)` 模式
- Rollup 将 `withRouteGuard` 错误解析到 common.js，运行时找不到定义 → ReferenceError → React 渲染树崩溃 → 白屏

### 修复方案
将所有 28+ 个页面的 HOC 模式改为内联 JSX 组件模式：

```diff
- import { withRouteGuard } from '@/components/RouteGuard'
- export default withRouteGuard(UserPage)
+ import { RouteGuard } from '@/components/RouteGuard'
+ return (<RouteGuard>{/* 原有内容 */}</RouteGuard>)
+ export default UserPage
```

### 修改文件
- `src/components/RouteGuard.tsx` — 标记 HOC 为 deprecated
- 28 个页面：user, address, admin*(4), cart, commission-detail, favorites, coupon, footprint, merchant*(7), my-promotion, payment, order-center, refund-apply, review, settings, ugc-publish, withdraw

### 技术教训
Taro 小程序中避免使用模块级 HOC 函数调用作为导出值，Rollup code-splitting 可能导致跨 chunk 函数引用丢失。改用 JSX 内联组件方式安全。

## 15. 门店页空白真凶：Rollup tree-shaking 掉了 common.js 和 vendors.js（第5轮，最终修复）

### 问题现象
经过前4轮修复（移除 useAuth/移除 useShareWithReferral/改用 inline style/零依赖测试版），门店页仍然空白：
- 导航栏标题始终显示硬编码"门店"（代码根本没执行）
- 内容区域完全空白
- 控制台有红色报错

### 真正的根因（构建产物级对比发现）

**Rollup tree-shaking 把 `common.js` 和 `vendors.js` 给干掉了！**

| 对比项 | store-home（空白） | explore/cart（正常） |
|--------|------------------|--------------------|
| `require("taro.js")` | ✅ | ✅ |
| `require("common.js")` | ❌ **缺失！** | ✅ |
| `require("babelHelpers.js")` | ✅ | ✅ |
| `require("vendors.js")` | ❌ **缺失！** | ✅ |

因为第4轮做了"零依赖测试版"，页面不 import 任何来自 `@/db/api` 的函数 → Rollup 认为不需要 `common.js` → 把它 tree-shaking 掉了 → 同时 `vendors.js` 也被移除 → 小程序运行时缺少必要代码 → **页面在框架层面就崩溃了**，组件函数根本不会执行。

### 最终修复
恢复 `getStoreById`、`getStoreCategories`、`getProducts` 的导入（来自 `@/db/api`），强制 Rollup 保留 `common.js` 和 `vendors.js`。

### 编译验证
✅ built in 16.05s，零错误
✅ store-home/index.js 现在包含全部 4 个 require：taro.js + **common.js** + babelHelpers.js + **vendors.js**
✅ JS 大小：6.46kB（包含完整功能）

### 技术教训
**Taro 小程序中每个页面必须至少导入一个来自项目内部模块的函数/组件**，否则 Rollup 会把 `common.js`（共享业务代码）和 `vendors.js`（第三方库运行时）tree-sh掉，导致小程序运行时崩溃。这不是代码逻辑问题，是**构建配置层面的陷阱**。
