# 来电有喜 · 会员成长段位重复与改造概览

> 本文记录「会员成长段位」相关代码从「多页零散重复」收敛为「统一常量 + 统一工具」的改造过程。
> 对应提交：`744631a`（去重）、`3f72b65`（主题 token 收口）、`a1e4a75`（品牌主色收口 + user 页段位色修复）、`b935d4b`（全页评审 + 重复审计收尾）。

## 1. 问题背景
会员成长段位的展示与判定逻辑（段位名称、阈值、配色、进度文案等）原本散落在多个页面各自定义：

- `src/pages/explore/index.tsx`
- `src/pages/index/index.tsx`
- `src/pages/food-scan/index.tsx`
- `src/pages/my-promotion/index.tsx`
- `src/pages/scan-result/index.tsx`
- `src/pages/rank-rules/index.tsx`
- `src/components/RankProgress/index.tsx`

同时「扫码」相关的工具函数也分散在多处，存在多处近乎相同的重复片段。
这导致：改一段位规则要改 N 个地方、各页配色口径不一、维护成本高、易回归。

## 2. 改造方案
### 2.1 段位常量收口
新增 `src/constants/ranks.ts`，集中定义：
- 段位枚举 / 名称 / 升级阈值 / 配色 token 引用
- 段位计算与展示的纯函数

各页面改为从 `ranks.ts` 引用常量与函数，删除页内重复定义（净减约 65 行重复）。

### 2.2 扫码工具收口
新增 `src/utils/scan.ts`，统一封装扫码（商品码 / 配料码）的解析与分发逻辑，
替代此前散落在页面里的扫码片段，避免重复与口径漂移。

### 2.3 配套：主题 token 收口（让段位色口径统一）
- `3f72b65`：把硬编码卡片样式收口为统一 token；补齐三态组件（EmptyState / PageHeader / Skeleton）基础 UI。
- `a1e4a75`：把品牌主色硬编码 `#A8552E` 全部替换为 `hsl(var(--primary))`；
  修复 `src/pages/user/index.tsx`（会员中心）中段位色引用错乱的问题，使段位配色与全局主题一致。

## 3. 收益
- 段位规则单点维护，改一处全局生效；
- 各页段位配色口径统一，杜绝「同段位不同色」；
- 减少重复代码约 65 行，降低回归风险；
- 为后续会员体系扩展（段位权益、成长任务）奠定统一地基。

## 4. 验证
- `tsgo -p tsconfig.check.json` 类型检查无新增错误（剩余为历史 deep 类型债，不影响运行）；
- `taro build --type weapp` 通过（EXIT=0），主包 < 1.5MB；
- 已在微信开发者工具真机核对会员中心 / 段位展示配色一致。
