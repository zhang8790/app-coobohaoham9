# 情绪编译 v3.1 · AI 卖点标题引擎

> 本次为「情绪编译工作台」的增量增强：给每条智能生成的文案配一句**卖点短标题**，让商家一眼分辨每条文案的情绪角度，并能一键采用为商品标题。

## 改了什么

### 1. 标题生成引擎（`src/utils/emotion-description.ts`）
- 新增 `generateEmotionHeadline(product, moodTags, sceneTags, category, variant)`
  - 高频情绪（治愈/满足/温馨/甜蜜/浪漫/惊喜/清爽/温暖/放松/活力/精致/怀旧/分享/送礼/舒适）走**语义定制短标题池**，最自然。
  - 其余情绪走「emoji + 意境词」通用模板兜底。
  - 确定性：相同输入 + variant → 同一标题；不同候选（variant 0/1/2）自然呈现不同情绪角度。
- 新增 `generateEmotionCopyList(...)`：返回 `{ headline, body }[]`，标题与正文同序对应。

### 2. 商家工作台（`src/pages/merchant-emotion-compile/index.tsx`）
- 编译完成后，自动为该段候选生成「AI 卖点标题」`curHeadline`。
- 编译结果卡片内展示：`🏷️ {卖点标题}` + 「采用」按钮，点击即把该标题填入上方「标题」输入框。
- 「🔄 换一版」切换候选时，卖点标题同步切换，角度差异一目了然。

## 效果示例
- 治愈类商品 → `🩹 被温柔接住的时刻` / `🩹 一身倦意，慢慢化开`
- 满足类商品 → `🍽️ 一口就松下来的满足` / `🍽️ 刚刚好，不多不少`
- 温暖类商品 → `🔥 从手心暖到心底` / `🔥 一点暖意，足以御寒`

- 产物指纹扫描确认「采用」「被温柔接住的时刻」等定制字符串已编入 `dist/pages/merchant-emotion-compile/index.js` 与公共 chunk。

## 合规
- 标题与正文均沿用既有合规口径：功效措辞=传统食养参考，禁用医疗宣称/绝对化用语，无新增违规风险。
