## 一、三端架构对齐（已正确）

| 端 | 入口 | 读写的主表 |
|---|---|---|
| 用户端 | 小程序 `explore/store-home/product/emotion-*` | `products`(is_active=true) / `product_emotion` / `orders` |
| 商家管理端 | 小程序 `merchant-*` + 网页版 `admin-web/merchant/*` | `products`(按 store_id) / `product_emotion`(按 product_id) / `stores` |
| 总管理后台 | 网页版 `admin-web`（role=admin） | `products`(review_status='pending' 审阅) / `stores` / `withdrawals` |

- **共表共字段**：三端全部指向同一套 Supabase 表，`store_id`/`product_id` 关联一致。
- **RLS 实际关闭**：`products`/`stores`/`orders` 在 00018/00021/00023/00028 多处 `DISABLE ROW LEVEL SECURITY` 且无后续重开 → 网页版用 anon key 写库**不被 RLS 拦截**，这是衔接有利条件。

## 二、已修复的衔接断点

### 🔴 断点 1：00070 迁移表名写错（复数）→ 食养双轨断裂
- **现象**：小程序商家情绪编译页保存食养标签时报 `column shiyang_tags does not exist`；用户端 `emotion-check` 永远读不到食养数据。
- **根因**：`00070_shiyang_ingredients.sql` 用 `ALTER TABLE public.product_emotions`（复数），但真实表名是**单数** `product_emotion`（00040 创建）。对不存在的表 ALTER 必失败 → `shiyang_tags`/`shiyang_copy` 从未加上。
- **修复**：已把 `supabase/migrations/00070_shiyang_ingredients.sql` 与 `deliverables/migration_00070_shiyang_standalone.sql` 表名改回单数，`ADD COLUMN IF NOT EXISTS` 保证幂等可重跑。
- **⚠️ 须你本机执行**（沙箱无 Supabase CLI）：
  ```sql
  -- Dashboard SQL Editor 粘贴 migrations/00070 全文 Run，或 supabase db push
  -- 跑完自校验：
  SELECT column_name FROM information_schema.columns
  WHERE table_name='product_emotion' AND column_name IN ('shiyang_tags','shiyang_copy'); -- 期望 2 行
  ```

### 🔧 断点 2：商品审阅流写库依赖隐藏触发器（已加固）
- **原状**：`products` 审阅完全依赖 00009 两个触发器（插入强制 `pending`+`is_active=false`；审核 `approved`→`is_active=true`）。应用层 `createProduct` 只写 `is_active:true`、`approveProduct/rejectProduct` 只写 `review_status`，触发器缺失则审阅流断裂。
- **修复（去触发器依赖，行为不变）**：
  - 小程序 `src/db/api.ts` `createProduct` → 显式 `review_status:'pending', is_active:false`
  - 网页版 `admin-web/src/api/admin.ts` `approveProduct` → 加 `is_active:true`；`rejectProduct` → 加 `is_active:false`
- **效果**：触发器在时重复设置无害；触发器万一缺失，三端审阅/上架仍能自洽。

## 四、仍存在的差异与注意（非阻断，建议知悉）

1. **食养 UI 仅小程序有**：网页版 `EmotionStudio` 只读 `dimension_tags`、不管理 `shiyang_tags`。食养打标只能在商家小程序端做——功能覆盖差，但非 bug，三端数据仍互通。
2. **网页版保存会重置审核态**：`EmotionStudio` 保存恒写 `review_status:'draft'`。若小程序端已 `submitted`，网页版保存会把它退回 `draft`（轻微回退，未改）。
3. **部署依赖**：食养列（00070）+ 审阅触发器（00009）须已部署到云端。沙箱无 CLI，请在你本机确认 `supabase db push` 已覆盖这两项，或用 Dashboard 粘贴全文 Run。
