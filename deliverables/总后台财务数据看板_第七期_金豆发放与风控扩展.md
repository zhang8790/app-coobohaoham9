> 收尾「🟡 代码层残缺」三项的最后一期。前六期已建好看板/会员/订单/三流水/CSV导出/环形图/金豆闭环/智能风控（3 规则）。本期把第六期「金豆累计发放」KPI 真正能填充、风控扩到 5 规则、看板轮询升级 Realtime。

## 一、本次落地内容

### 1. 金豆后台发放 / 扣减入口（🟡1）
- **`admin-web/src/api/finance.ts`** 新增 `adminAdjustGoldBean(userId, amount, direction, reason)`：
  - 读 `profiles.gold_beans` 当前余额 → 计算 `delta`（grant 取正 / deduct 取负）→ `balance_after = max(0, cur+delta)`；
  - 写 `gold_bean_logs`（type=`admin_grant`/`admin_deduct`，带 `balance_after` 与 `remark`）；
  - 同步 `profiles.update({ gold_beans: balanceAfter })`；
  - 两步任意失败返回结构化 `{ ok, error }`，不抛异常、不阻塞调用方。
- **`admin-web/src/pages/Members.tsx`** 会员档案抽屉新增「💎 调整金豆」区块：
  - 发放/扣减切换按钮、金额输入框（1 金豆=1 元）、原因输入框；
  - 提交后 toast 反馈（成功显示新余额），自动刷新档案 + 列表；
  - 扣减设下限 0，不会把余额打负。
- **价值**：第六期「金豆累计发放」KPI 此前只会被退款（`refund_return`）填充，现在平台可主动发放/扣减，闭环成立。

### 2. 智能风控规则扩展（🟡2）
`getAnomalyReport()` 由 3 条扩到 5 条，全复用 PostgREST 聚合 + 两步直读，零未定义 RPC：
- **R1 门店异常退款率**（原）：成交≥5 单且退款率 >30%→关注 / >50%→高危；
- **R2 单日 GMV 骤降**（原）：较昨日跌 >50% 且今日有成交；
- **R3 大额订单关注**（原）：近 20 笔中 >¥5000；
- **R4 封禁会员占比异常（新）**：`count(profiles.is_banned=true)/count(profiles)` >5%→关注 / >15%→高危（疑似批量封禁或风控失效）；
- **R5 刷确权异常（新）**：`emotion_claims` 按 `user_id` 聚合，单用户累计确权 >20 次 或 近 24h >5 次 → 关注（疑似刷量）。
- 看板「智能风控（异常检测）」区块自动呈现，每刷新重算。

### 3. 看板 Realtime 升级（🟡3）
- **`admin-web/src/pages/FinanceDashboard.tsx`**：
  - 引入 `supabase` 客户端，`useEffect` 内 `supabase.channel('finance-dashboard')` 订阅 `orders`/`profiles`/`gold_bean_logs`/`emotion_claims` 的 `postgres_changes`（`event:'*'`），任意写入即 `refresh()`；
  - 保留 **60s 兜底轮询**（防 Realtime 未启用时数据陈旧）；
  - 卸载时 `supabase.removeChannel(channel)` + `clearInterval`，无泄漏；
  - 区块说明文案改为「实时订阅更新（兜底 60s 轮询）」。

## 三、⚠️ 必须你本机执行的动作（沙箱无 Supabase CLI / 无真浏览器）
1. **跑迁移**：`00070`(食养列) / `00072`(emotion RLS) / `00073`+`00074`(emo 徽章+body_templates) / `00075`(points_logs 放开 RLS) / `00076`(gold_bean_logs 建表) —— 不跑则积分流水、金豆流水后台为空，金豆发放写入也会被 RLS 挡。
2. **开启 Realtime 发布（本期新增依赖）**：在 Supabase Dashboard → Database → Replication 把 `orders` / `profiles` / `gold_bean_logs` / `emotion_claims` 加入 `supabase_realtime` 发布；否则看板走兜底 60s 轮询（功能不变，只是非实时）。
3. **金豆发放写入权限**：`adminAdjustGoldBean` 用 anon key 直写 `gold_bean_logs` + 更新 `profiles.gold_beans`。需确认 `gold_bean_logs` RLS 已按 00076 关闭（已 GRANT anon）、且 `profiles` 当前 RLS 状态允许 anon update（本项目历史多表 RLS 关闭，若 profiles 已开 RLS 且无 admin policy，更新会被拒——届时报错「余额更新失败」但流水已记，按提示人工核对，或加 `policy(get_user_role(auth.uid())='admin')`）。
4. **核对**：`cd admin-web && npm run dev` → admin 进 `/members` 开某会员档案点「💎 调整金豆」发放 100，确认该会员金豆 +100、看板「金豆累计发放」+100、金豆流水 tab 出现 `admin_grant` 行；进 `/finance` 看风控区块在异常数据下是否触发 R4/R5。
