# 来电有喜 · 本机执行手册（数据库迁移 + 云函数 + 实时订阅 + 验收）

> **为什么需要本机执行**：沙箱环境无 Supabase CLI、无真机浏览器，下列动作无法在沙箱自动完成。
> **代码层状态**：小程序 + admin-web 全部功能已实现，并通过产物级验证（TSC / Vite / Taro 全部 EXIT=0）。
> 本手册把剩余 ⚠️ 项变成一份可照做的清单。

---

## 一、前置准备

1. 安装 Supabase CLI：`npm i -g supabase`
2. 登录：`supabase login`
3. 链接项目：`supabase link --project-ref <你的 project-ref>`
   （project-ref 在 Supabase Dashboard → Project Settings → General）
4. 确认本地 `src/client/supabase.ts` 的 URL + anon key 与线上一致

---

## 二、执行数据库迁移（6 个文件，按顺序）

位于 `supabase/migrations/`：

| 文件 | 作用 | 不执行后果 |
|------|------|-----------|
| `00070_shiyang_ingredients.sql` | 食养食材表 + 商品食养标签列 | 食养打标功能无数据 |
| `00072_fix_emotion_rls.sql` | 关闭 emotion_* 表 RLS（客户端直插） | 情绪确权 403 |
| `00073_seed_emo_badge_defs.sql` | 补 9 个 emo_* 情绪态徽章 | 发徽章 409 外键拒绝 |
| `00074_add_body_templates.sql` | category_emotion_profiles 加 body_templates 列 | 前端 select 该列 400 |
| `00075_disable_points_logs_rls.sql` | 放开 points_logs RLS | 后台积分流水空白 |
| `00076_create_gold_bean_logs.sql` | 建 gold_bean_logs 表 + 关 RLS | 后台金豆流水空白 |

**执行方式（任选其一）**

- CLI 推全部未执行迁移：
  ```bash
  supabase db push
  ```
- 或单文件执行（psql）：
  ```bash
  psql "$DATABASE_URL" -f supabase/migrations/00076_create_gold_bean_logs.sql
  ```
- 或 Dashboard：SQL Editor 粘贴文件全文 → Run

> ⚠️ **红线**：`00072 / 00075 / 00076` 均含 `DISABLE ROW LEVEL SECURITY`。执行后**切勿在 Dashboard 手动 Enable RLS**（无 policy 会再次 403 / 400）。

## 四、开启 Realtime 发布（第七期看板实时刷新依赖）

- `orders`
- `profiles`
- `gold_bean_logs`
- `emotion_claims`

> 未开启时看板走 60s 兜底轮询（功能正常，仅非实时）。

## 五、确认 profiles RLS（金豆后台发放依赖）

`admin-web` 的 `adminAdjustGoldBean` 用 anon key 直写 `profiles.gold_beans`。

- 若 `profiles` RLS **开启**：需加 policy 允许 anon / service_role 更新自己的行，否则报「余额更新失败」（流水已记，按提示人工核对或加 admin policy）。
- 校验：Dashboard → Authentication → Policies 查看 profiles 策略。

## 六、启动 admin-web 核对

```bash
cd admin-web && npm install && npm run dev
```

1. `/finance` 资产流通区：**金豆余额 / 金豆累计发放 / 金豆累计消耗** 三卡

3. `/ledgers` 四 tab（积分 / 情绪豆 / 佣金 / 金豆）均有数据
4. 导出 CSV 用 Excel 打开，确认中文不乱码（已带 UTF-8 BOM）
