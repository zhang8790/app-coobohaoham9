## 步骤 1 · 跑迁移 00084（Dashboard SQL Editor）

Dashboard → SQL Editor → New Query → 粘贴运行：

```sql
-- 00084: 通知中心 notifications 表
CREATE TABLE IF NOT EXISTS notifications (
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      uuid NOT NULL,
  type         text NOT NULL,
  title        text NOT NULL,
  body         text NOT NULL,
  order_id     uuid,
  payload      jsonb NOT NULL DEFAULT '{}'::jsonb,
  read_at      timestamptz,
  sent_at      timestamptz,
  created_at   timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_notifications_user_unread_created
  ON notifications (user_id, created_at DESC) WHERE read_at IS NULL;
CREATE INDEX IF NOT EXISTS idx_notifications_user_created
  ON notifications (user_id, created_at DESC);

ALTER TABLE notifications DISABLE ROW LEVEL SECURITY;

COMMENT ON TABLE notifications IS '通知中心：所有推送给小程序用户的业务事件';
COMMENT ON COLUMN notifications.type IS 'order_paid | commission_arrived | withdraw_progress | refund_result | announcement';
```

**验证**：

```sql
SELECT count(*) FROM notifications;  -- 应为 0（刚建表）
SELECT column_name FROM information_schema.columns WHERE table_name='notifications' ORDER BY ordinal_position;
-- 应返回 9 行：id, user_id, type, title, body, order_id, payload, read_at, sent_at, created_at
```

## 步骤 5 · 验证上线（anon 探针）

```bash
# send-notification 单发：缺 openid 时应返回 success:true, sent:false, reason:no_openid
curl -s -X POST "https://<project>.supabase.co/functions/v1/send-notification" \
  -H "apikey: <anon-key>" -H "Content-Type: application/json" \
  -d '{"user_id":"00000000-0000-0000-0000-000000000000","type":"order_paid","title":"测试","body":"测试内容"}'

# send-notification 广播（公告）：应返回 {success:true, mode:'broadcast', results:{total:..., sent:..., failed:..., skipped:...}}

  -H "apikey: <anon-key>" -H "Content-Type: application/json" \
```

## 可能踩的坑

1. **wx.requestSubscribeMessage 必须在「点击事件」中调用**，不能在 `useEffect` 里直接弹。我们放在支付/分佣/提现的回调里，OK。
2. **用户拒绝后当天不能再弹**，所以 `useSubscribeMessage.isInCooldown()` 默认 7 天冷却。
3. **notifications 表无 FK 指向 auth.users**：因为现有 `commissions/withdrawals` 都是这种风格（避免破坏 RLS 配置）。测试期安全。
4. **profiles.openid 为空时通知会写库但不推送**：用户进入消息中心仍能看历史。
5. **tmpl_id 申请 1-3 个工作日审核**，趁早申请。
6. **微信订阅消息每次推送都需用户授权一次**——这是微信规则，不是 bug。开发者无解。
