# 资金安全修复 · 本机一键执行脚本

> 沙箱（WorkBuddy）**无法部署**：supabase CLI 在 win32 下无可用二进制 + 无 `SUPABASE_ACCESS_TOKEN`。
> 以下命令需你在**自己本机**（已装 supabase CLI + 已 `supabase login` + 项目已 `link`）执行。
> 本文件是「交付说明」的执行版：顺序已排好，复制即可跑。

## 0. 前置（一次性）

```bash
# 安装 CLI（若未装）
npm i -g supabase

# 登录（浏览器授权，拿到 token 缓存）
supabase login

# 进入项目根目录
cd /path/to/app-coobohaoham9

# 关联远程项目（若之前没 link 过；ref 在 Supabase 后台 Settings → Project ID）
supabase link --project-ref <你的PROJECT_REF>

# 确认云端环境变量已设（deploy 函数时需要；按各函数 Deno.env.get 的名字）
supabase secrets list
# 常见需有：SUPABASE_URL、SUPABASE_SERVICE_ROLE_KEY、MCH_API_V3_KEY、WECHAT_*
# 缺哪个补哪个： supabase secrets set MCH_API_V3_KEY=xxx
```

## 1. 跑迁移（按文件名顺序，只应用未执行过的）

**推荐用 CLI（自动比对 `supabase_migrations` 表，幂等安全）：**

```bash
supabase db push
```

### 本次新增的迁移
- `00080_atomic_rpc_indexes_unique.sql` —— **现在必须跑**（止血用：原子 RPC + 索引 + emotion_claims 唯一**索引**）。
  - 已内置「先 DELETE 去重再加 UNIQUE」，即使表里有历史重复确权行也不会让迁移失败。
  - ⚠️ **已修复**：原 `ADD CONSTRAINT IF NOT EXISTS ... UNIQUE` 在 Supabase SQL Editor 报 42601 语法错，已改为 `CREATE UNIQUE INDEX IF NOT EXISTS`（效果等价、百分百兼容）。
- `00081_production_rls_hardening.sql` —— **全注释模板，测试期建议先不跑**，生产前再取消注释执行（见第 4 步）。

### 早期遗留未跑的迁移（若此前没在 Dashboard 手动执行过，db push 会一并应用）
`00054` `00072` `00073` `00074` `00076` `00077` `00078`
> 这些早已是 `IF NOT EXISTS` / `CREATE OR REPLACE` 幂等写法。
> 若你之前已在 Supabase SQL Editor 手动跑过其中某些（比如手建过 gold_bean_logs 表），`db push` 可能因「已存在」报错——
> 解法：在 SQL Editor 执行 `INSERT INTO supabase_migrations (version) VALUES ('00076');`（填对应版本号）标记为已应用，再重跑 `supabase db push`。

### 备选：Dashboard 粘贴（不用 CLI 也行）

## 3. 验证提现链路真跑通

```sql
-- (1) 制造一笔新支付 / 纯金豆订单后，看受益人可提现余额是否增加
SELECT id, commission_balance, total_commission
FROM profiles
WHERE id = '<受益人user_id>';

-- (2) 看 commissions 是否有对应分佣记录
SELECT * FROM commissions ORDER BY created_at DESC LIMIT 5;

-- (3) emotion_claims 唯一索引是否建上（00080 的副产物，用唯一索引代替约束以兼容 SQL Editor）
SELECT indexname
FROM pg_indexes
WHERE tablename = 'emotion_claims' AND indexname = 'uq_emotion_claims_user_order';
```

预期：① `commission_balance` 在订单完成后变大；② `commissions` 出现新行；③ 查到 `uq_emotion_claims_user_order` 唯一索引。
