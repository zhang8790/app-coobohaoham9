# 微信订阅消息 · 模板申请清单

> 用途：本项目接入微信官方「订阅消息」能力，让用户收到「订单/分佣/退款/提现/公告」推送。  
> 本文件告诉你**去 mp.weixin.qq.com 后台申请 5 个模板时该填什么字段**。  
> 申请后把 5 个 tmpl_id 回填到 Supabase Edge Function 环境变量。

---

## 登录路径

1. 打开 https://mp.weixin.qq.com/
2. 用小程序管理员微信扫码登录
3. 左侧菜单 → **订阅消息** → **公共模板库** 或 **我的模板** → **添加**
4. 搜索关键词，选中模板 → 选「行业-工具-效率」类 → 提交

---

## 环境变量总览（粘贴到 Supabase Dashboard → Edge Functions → send-notification → Settings → Secrets）

```bash
# 已有
WX_APPID=你的小程序 appid
WX_SECRET=你的小程序 appsecret

# 新增（5 个 tmpl_id）
TMPL_ORDER_PAID=
TMPL_COMMISSION_ARRIVED=
TMPL_WITHDRAW_PROGRESS=
TMPL_REFUND_RESULT=
TMPL_ANNOUNCEMENT=
```

## 字段类型说明

微信订阅消息字段分 4 类（必须选对）：

| 类型 | 用途 | 示例 |
|---|---|---|
| `thing` | 20 字以内文本 | 订单号、商品名 |
| `amount` | 金额 | ¥10.00、¥1,234.56 |
| `time` | 时间 | 2026-07-11 12:00 |
| `phrase` | 短语枚举 | 已发货、待付款、已退款 |

注意：thing 字段会**自动截断 20 字**，所以我们后端都做了 `.slice(0, 20)`。

---

## 订阅消息的规则（用户视角）

1. **用户必须主动授权**：每次弹窗都需用户点「允许」，可单次选 1-5 个模板
2. **长期有效**：用户授权 1 次后，开发者**每次**推送前都需用户重新授权（一次性）
3. **拒绝冷启动**：用户点「拒绝」后，开发者**当天**不能再弹
4. **跳小程序**：模板可指定「用户点击通知 → 跳到哪个页面」，我们用 `page: 'pages/messages/index'`

我们的 `useSubscribeMessage` hook 已封装：
- 支付成功后 → 主动弹授权（仅 order_paid tmplId）
- 佣金到账后 → 主动弹（仅 commission_arrived）
- 提现发起后 → 主动弹（仅 withdraw_progress）
- 7 天 cooldown（拒绝后）

## 申请小贴士

- 「公共模板库」找不到完全匹配的，可在「我的模板」中**自定义申请**（需 1-3 个工作日审核）
- 字段顺序是固定的，**必须按本清单的顺序**，否则 send-notification 的 `buildWxData` 字段对不上
- 每个小程序**最多 25 个模板**，我们用 5 个远未触顶
- tmpl_id 申请后**不可修改字段名**，但可申请新模板替换
