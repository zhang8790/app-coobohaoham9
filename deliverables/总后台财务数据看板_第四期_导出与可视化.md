## 一、全站 CSV 导出对账

### 1.1 导出工具 `admin-web/src/lib/csv.ts`（新建）
- `toCSV(rows, columns)`：对象数组 → CSV 字符串，字段内逗号/引号/换行自动转义（`"xx"` 包裹 + 双引号翻倍）
- `downloadCSV(filename, rows, columns)`：带 **UTF-8 BOM**（`\ufeff`）触发浏览器下载 → **Excel 打开中文不乱码**
- `csvTimestamp()`：文件名后缀 `YYYYMMDD_HHMM`，避免覆盖

### 1.2 全量导出查询 `admin-web/src/api/finance.ts`（追加）
新增 4 个全量导出函数，复用已包装的列表查询（含两步直读昵称/手机）：
- `exportOrders(filters)` — 循环分页拉全量订单（按当前筛选）
- `exportPointsLedger(filters)` — 全量积分流水
- `exportEmotionLedger(filters)` — 全量情绪豆流水
- `exportCommissionLedger(filters)` — 全量佣金流水

内部用通用 `fetchAll(pageFn)` 循环分页（每页 500），直到拉完 `total`，**不会漏行也不会死循环**。

### 1.3 三处导出入口（均已进产物）
| 入口 | 文件 | 导出内容 |
|---|---|---|
| 财务看板「⬇ 导出 CSV」 | `FinanceDashboard.tsx` | 财务明细总表 8 行（GMV/让利/佣金/净收益/金豆/积分/情绪豆/CV + 占 GMV 比） |
| 成交订单「⬇ 导出全部」 | `Orders.tsx` | 当前筛选下全部订单（订单号/买家/手机/门店/成交额/让利/平台实收估/状态/退款/时间） |
| 资产流水中心「⬇ 导出全部」 | `Ledgers.tsx` | 当前 tab 全部流水（积分/情绪豆/佣金，按类型/状态/层级筛选） |

导出过程有 `exporting` 状态防重复点击，按钮显示"导出中…"。

## 二、看板智能化：SVG 环形饼图

### 2.1 新增 `DonutChart` 组件（手绘 SVG）
- `strokeDasharray` + `strokeDashoffset` 实现环形分段，中心显示总标签（支持两行文本）
- 纯 CSS/SVG，无第三方图表库依赖，契合项目 premium 暗色风格

### 2.2 两张智能图（看板中部）
1. **收益结构（GMV 去向）** — 同量纲（元），精确展示平台从 GMV 里：
   - 🟡 让利（gold） / 🟣 推广佣金（purple） / 🔵 平台净收益（blue）
   - 直观回答"钱去哪了"：让利挤压了多少、分给推广多少、平台真正落袋多少
2. **数字资产发行结构** — 金豆(1=1元)/积分/情绪豆/贡献值 CV 四资产占比总览（图例标注单位，诚实区分量纲）

## 四、需你本机做的（沙箱无 Supabase CLI）

1. `cd admin-web && npm run dev`，admin 账号登录
2. 访问 `/finance` → 点「⬇ 导出 CSV」下载财务明细，用 Excel 打开确认中文不乱码
3. 访问 `/orders` → 点「⬇ 导出全部」核对订单导出（先用筛选缩小范围验证）
4. 访问 `/ledgers` → 切到三张表分别点「⬇ 导出全部」核对流水导出
5. 确认 `00075` 迁移已执行（否则 points_logs 仍为 0 行，积分流水导出为空）
