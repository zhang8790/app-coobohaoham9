## 二、成交订单页（详细化）

**列表列**
- 订单号（无则取 ID 前 8 位）、买家（昵称+手机）、门店（无则「平台直营」）
- 成交额、让利（金豆抵扣）、**平台实收(估) = 成交额 − 让利**
- 状态（8 种枚举带色标：待支付/待发货/待收货/待核销/待评价/已完成/售后/已取消）、退款状态、下单时间
- 顶部本页汇总卡：本页成交额合计 / 让利合计 / 平台实收估

**详情抽屉**（点击行）
- 订单号、状态、门店、买家、买家手机、推荐人(上线)、成交额、让利、**平台实收(估)**、退款状态、下单时间
- 平台实收估算逻辑与看板 `platformNet` 一致（GMV − 让利），与 `commissions.commission_amount` 解耦，避免未知 schema 误算

**筛选**：状态下拉（全部/各枚举）+ 订单号模糊搜索 + 分页（每页 15）

## 三、智能化增强（数字化 → 智能化）

1. **自动刷新（活的数据）**
   - 首屏加载后启动 `setInterval(refresh, 30000)`，每 30s 重新聚合
   - 手动「⟳ 刷新」按钮 + 右上角「更新于 HH:MM:SS」时间戳
   - 刷新态禁用按钮防重复点击

2. **智能预警横幅**（基于真实聚合，非写死）
   - 🚫 `withdrawPending > 0` → 「有 N 笔提现待处理」（warn）
   - ⚠️ `让利率 = 让利/GMV > 15%` → 「让利率 X% 偏高（挤压平台净收益）」（danger）
   - 🚫 `bannedMembers > 0` → 「N 名会员被封禁」（warn）
   - 无预警时不渲染横幅（零噪音）

3. **多指标环比（数字化洞察）**
   - 近 7 日 vs 前 7 日，对 **GMV / 成交量 / 注册量** 三指标分别算环比 %
   - KPI 卡直接显示「↑/↓ X%（近7日）」：成交额 GMV、成交订单数、今日新增会员

## 四、与用户端对接（同源同口径）

订单页/看板读的就是**用户端小程序写入的同一套 Supabase 表**：
- `orders`（成交额/让利/状态）— 小程序下单即写入
- `profiles`（买家手机/昵称/上线/封禁）— 用户注册即写入
- `stores`（门店名）— 商家入驻即写入
- `withdrawals`（提现）— 用户/商家提现即写入
- `commissions`（佣金，看板平台净收益已用 `settled` 态）

用户在小程序下单/消费/提现，**总后台即时可见、即时预警**。
