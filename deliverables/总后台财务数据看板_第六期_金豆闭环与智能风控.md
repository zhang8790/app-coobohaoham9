## 二、金豆总览闭环

### 数据底座（复用已验证范式）
- `FinanceOverview` 接口新增 `goldBeanIssued` / `goldBeanConsumed` 两个字段。
- `getFinanceOverview` 内按 `gold_bean_logs.type` 做定向聚合（`sumOf` 支持 `eq` filter，无需拉全表、无需 RPC）：
  - **累计发放** = `refund_return` + `recharge` + `admin_grant` 的正向 delta 之和
  - **累计消耗** = `|purchase_spend|` + `|admin_deduct|` 的负向 delta 绝对值之和
- `gold_bean_logs` 未建（00076 未执行）时，`sumOf` 捕获错误降级为 `0`，**不阻塞整页**；金豆余额仍取自 `profiles.gold_beans`。

## 三、智能风控异常检测引擎

### 聚合层（`finance.ts` 新增 `getAnomalyReport`）
零未定义 RPC，全部复用 PostgREST 聚合 + 两步直读范式，单规则失败降级为不触发。

| 规则 | 逻辑 | 触发条件 | 等级 |
|---|---|---|---|
| **R1 门店退款率异常** | 拉 `orders(store_id, refund_status)` 全量，JS 按店聚合；排除 `refund_status` 为 `none`/`null` 字面量 | 成交 ≥5 单 且 退款率 >30%（>50% 升高危） | 关注 / 高危 |
| **R2 单日 GMV 骤降** | `getDailyTrend(2)` 取今日 vs 昨日 | 今日/昨日 <50% **且今日>0**（防凌晨空窗误报） | 关注 |
| **R3 大额订单关注** | `orders` 按 `total_amount` 降序取近 20 笔 | 其中 >¥5000 的订单存在 | 提示 |

返回结构：
```ts
interface Anomaly { id: string; level: 'high'|'medium'|'low'; title: string; detail: string; metric?: string }
interface AnomalyReport { anomalies: Anomaly[]; checkedAt: string }
```

### 看板呈现（新增「智能风控（异常检测）」Section）
- 紧随资产流通区块，跨整行卡片。
- 每条异常带**等级色标**：高危=红、关注=橙、提示=蓝（与既有预警横幅风格一致）。
- 无异常时显示「✅ 未发现异常指标」。
- 底部标注「检测于 HH:MM:SS · 每 30 秒自动刷新」——随总览 `refresh` 一并重算，无需新增定时器。
