## 一、 商家管理中心（用户端小程序“掌柜阁”）

### 1.1 准入规则与角色体系
* **准入方式**：
  * 用户在前台通过扫专属“商户开通码”或者超级管理员后台“武林盟”手动授予“商家/掌柜”角色。
  * 用户登录后，在“我的”页面，若角色为商家，会自动展示“掌柜阁”或“切换掌柜端”按钮。
* **角色说明**：
  * **正掌柜（店主）**：拥有全部权限（含账目查看、员工授权、提现申请）。
  * **店小二（核销员）**：由掌柜通过“小二授权码”扫码绑定，仅拥有“订单核销”与“核销历史查看”权限，无权查看钱包明细、提现或上架新商品。

---

### 1.2 核心功能模块一：扫码上架（宝贝管理）
在用户端，商家无需复杂的电脑操作，可直接在手机端通过扫条形码或手动录入，在“掌柜阁”快速上架和管理店内商品（江湖宝贝）。

#### 1.2.1 交互流程
1. 掌柜点击掌柜阁首页的 **“上架宝贝”** 按钮。
2. 页面底部弹出选择框：
   * **[扫码识别商品]**：调用手机摄像头扫描商品条形码（EAN码），系统调用国家标准条码库API自动匹配商品名称、规格、图片；匹配成功则自动填充表单，不成功则提示“未知条码，请手动录入”。
   * **[手动录入商品]**：进入空白创建宝贝表单。
3. 表单填写完成后，点击 **“申请面世（提交审核）”**，信息送至管理后台。

#### 1.2.2 页面字段与按钮详情
| 字段/元素名 | 类型 | 输入规则与校验约束 | 说明（武侠包装） |
| :--- | :--- | :--- | :--- |
| **宝贝名称** | 文本 | 必填，最多30字。不允许出现敏感词。 | “秘籍雅号” |
| **宝贝分类** | 单选 | 必填。选项：灵丹妙药（餐饮）、绝世武功（预订服务）、神兵利器（零售周边） | “所属派别” |
| **原价** | 数字 | 必填，单位：两（元）。万分位精度，不能为负。 | “基础铜板” |
| **优惠价** | 数字 | 必填，须小于或等于原价。 | “结缘碎银” |
| **宝贝图片** | 图片 | 最多上传5张，首张默认为主图。大小不超过5MB。 | “秘籍丹青” |
| **宝贝介绍** | 多行文本 | 可选，最多300字。 | “秘籍真言” |
| **服务模式** | 多选 | 必选其一。选项：[堂食就座/到店服务]、[外卖自送/飞鸽传书] | “客官自提/飞鸽传书” |

* **按钮功能**：
  * **[暂存草稿]**：保存至本地，不提交审核，宝贝状态变为 `draft`。
  * **[上架/申请面世]**：提交数据，数据库创建商品记录，状态为 `pending_review`（待审核）。

### 1.3 核心功能模块二：订单核销（接头暗号）
本地生活预订与到店餐饮的核心交互环节。通过“接头暗号（核销码）”实现快速消费确认。

#### 1.3.1 核销流程设计
1. 顾客到店，在“我的订单”中展示订单的“接头暗号QR码”或“十二位核销纯数字”。
2. 掌柜或店小二打开掌柜阁首页，点击 **“扫暗号（扫码核销）”**，拉起摄像头扫描。
3. 扫描成功，触发以下逻辑校验：
   * **校验一（门店归属）**：该订单对应的商品是否属于当前门店？若不属于，报错：`此非本派门产，客官走错门了！`。
   * **校验二（订单状态）**：订单是否处于 `pending_ship`（待核销/待发货）状态？
     * 若已核销，报错：`此暗号已失效，莫非是江湖骗子？`。
     * 若未付款，报错：`此单未付清茶钱，无法核销！`。
4. 校验通过，页面展示订单详情（商品、数量、实付金额、买家昵称）。
5. 掌柜确认无误后，点击 **“对暗号（确认核销）”** 按钮。状态更新为 `completed`（已完成），资金正式结算至商家未提现余额。

#### 1.3.2 页面元素与按钮详情
* **[手动对暗号（数字核销）]** 链接：
  * 点击弹出输入框，支持手动输入 12 位数字暗号，同样执行上述校验。
* **[确认核销]** 按钮：
  * 触发后端安全核销事务：更改订单状态为 `completed` $\rightarrow$ 扣减库存 $\rightarrow$ 累加商家余额。

### 1.4 核心功能模块三：掌柜钱包与提现（镖银运送）
用于查看商家的总营业额、未提现余额及发起提现。

#### 1.4.1 提现基础规则（“镖银运送”法则）
* **提现额度**：最低提现额度为 **100元**。
* **手续费（江湖税）**：提现扣除 **10%的手续费**（用于覆盖微信支付商户通道费及系统维护成本）。
* **提现方式**：微信零钱直接代付（单笔1000元以下且审核通过）或人工银行转账。
* **结算期**：订单核销后（即状态变为 `completed`），资金立刻计入商家余额，随时可提现。

#### 1.4.2 提现交互流程
1. 掌柜进入“掌柜钱包”，查看【未提提现额】、【已提现总额】。
2. 点击 **“运送镖银（申请提现）”**，进入提现页面。
3. 输入提现金额。系统实时在下方显示：`扣除10%江湖税：X元，预计实到：Y元`。
4. 确认后，点击 **“拉镖启程（确认提现）”**，发起提现请求，生成提现单，状态变更为 `pending_audit`。

---

### 1.5 核心功能模块四：门店公告与经营管理
用于管理门店的基础信息和前台展示风格。

#### 1.5.1 经营状态挂牌/摘牌
* **[挂牌营业]（置为上线）**：前台商品可被搜索、可见、可购买。
* **[闭门谢客]（置为下线）**：商品在前台依然显示，但“立即购买”按钮变灰并显示“闭门谢客中，明日请早”。

#### 1.5.2 门店公告配置
* 掌柜在“公告编辑”框输入例如：`今日小店新进一批百年女儿红，欢迎各位少侠品尝！`（最大50字）。
* 公告保存后，前台门店首页及商品详情页头部滚动播放。

## 二、 超级管理后台（PC/全局端“武林盟”）

超级管理后台为平台方提供对整个武林生态的绝对掌控力。

### 2.1 商家审核与准入管理（门派大典）
对用户提交的开店、上架商品申请进行集中审核。

#### 2.1.1 商家入驻审核列表
* **审核条件**：审查商家的“门派掌门身份”（身份证/营业执照/卫生许可证等）。
* **审核按钮**：
  * **[通过（准许开山立派）]**：生成 `store` 记录，自动为其生成专属的 8 位短码（`short_code`）和门店二维码，用户前台角色变更为 `merchant`。
  * **[驳回（逐出山门）]**：点击必须弹出对话框，输入“驳回理由”（例如：资质不全、敏感词等），通知通过小程序消息送达。

#### 2.1.2 宝贝上架审核
* **过滤机制**：敏感词过滤，对暴力、政治、虚假宣传进行审核。
* **审核按钮**：
  * **[批准面世]**：商品状态变更为 `published`，商家可挂牌销售。
  * **[扣押不发]**：商品变更为 `rejected` 状态，退回商家草稿箱并附带驳回原因。

---

### 2.2 提现审核与风控流（银票兑付）
为了平台的现金流安全，所有提现（无论是用户分佣提现，还是掌柜营业额提现）均在此处审核。

#### 2.2.1 提现风控规则
* **自动拦截线**：单笔提现金额 > 500元，或者日累计提现 > 1000元，系统强行标记为“需高级人工审核”，不予执行自动代付。
* **防套利校验**：自动校验发起提现的账户最近7天内的核销记录与佣金增加记录。若发现“空订单核销（核销后1分钟内即发起大额提现）”，系统产生【套利红字警报】。

#### 2.2.2 提现单管理表格按钮详情
* **[标记为异常（打入黑名单）]**：冻结该提现单，商家账号限制提现15天，需提交线下经营证明。
* **[手动出款（线下打款）]**：线下通过网银转账后，管理员点击此按钮，上传转账截图，订单变更为 `completed`（已完成）。
* **[一键代付（微信直发）]**：调用微信支付代付接口直发至用户微信钱包。

---

### 2.3 用户段位与动态分佣配置（功力天梯）
本模块允许平台管理员根据运营策略，实时调整不同段位的分佣比例、锁定比例等全局常数。

#### 2.3.1 配置表单字段详情
* **让利池基准比例**：默认 25%。(可微调，影响全平台商品分佣池总额)。
* **各段位分佣比例配比表**：
  *(支持配置 1~6 级段位的 L1 比例与 L2 比例)*
  * 江湖散修（L1: 30%, L2: 12%）
  * 长老（L1: 54%, L2: 24%）
  * 掌门（L1: 60%, L2: 27%）
* **全局佣金封顶线**：默认 60%（占让利池），防亏损关键常数，可在线调整为 50%-75%。
* **积分返现比例**：默认 25%（占让利池），支持调整。
* **每日积分上限**：默认 200分（2.0元等值），超过后当日不再返积分。

---

### 2.4 UGC 内容审核与舆情监控（武林贴管理）
审核用户发表的情绪贴、评价贴、游记（UGC内容），防止违规舆情。

#### 2.4.1 舆情审核功能点
* **[置顶（封为武林盟主推荐贴）]**：点击后，该贴在前台首页获得显眼“精品/精选”标签，并推至推荐流顶部。
* **[敏感词高亮]**：自动扫描发文内容。若含违禁词，高亮标记并在列表头部展示。
* **[一键禁言/删帖]**：
  * 点击 **“删帖（销毁秘籍）”**：逻辑删除内容。
  * 点击 **“禁言（封印功力）”**：封禁发帖用户 7-30 天发帖权限。

---

### 2.5 退款与纠纷仲裁（调解恩怨）
处理买家与商家的交易纠纷。

#### 2.5.1 退款审批规则
* **未消费退款**：对于未核销、未发货订单，若买家发起退款：
  * 若是餐饮/普通服务类，支持 7 天无理由退款，商家同意后平台全额退还（包括扣减已分销的冻结佣金，防退款套利）。
* **已消费纠纷**：买家对已核销商品发起投诉并要求退款。
* **后台仲裁按钮**：
  * **[强制退款（仲裁归买家）]**：原路退回付款，商家钱包余额中扣减相应金额。原分佣已发出的部分，从分销商的未提现钱包中直接扣减（若其余额不足，显示为负数，并在下次分佣中自动冲抵）。
  * **[驳回申诉（仲裁归掌柜）]**：支持商家不退款，订单维持 `completed` 状态，纠纷单平账。

## 三、 全流程交互流程图（Flowcharts）

### 3.1 扫码核销异常处理交互流

```
       [开始: 店小二/掌柜点击扫暗号]

              [拉起摄像头扫码]
                     │
              [校验订单合法性]
              /              \
     [合法: 属于本派且待消费]   [异常: 走错门/已失效/未付钱]
             │                                │
     [展示订单详情确认页]               [高频红色警报震动提示]
             │                                │
     [掌柜点击"确认核销"按钮]            [弹出弹窗显示具体报错信息]
             │                                │

             │                                │
     [变更状态为"completed"]                  [结束流]
             │
     [资金入账掌柜余额, 触发佣金结算]
             │
          [结束流]
```
