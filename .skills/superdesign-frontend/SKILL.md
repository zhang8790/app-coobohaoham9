## 能力概述

本技能为 Agent 提供一套结构化、现代化的前端 UI 设计方法论，帮助生成美观、可用、无障碍的网页界面。

| 属性 | 值 |
|------|-----|
| 技能层级 | `helper-tier`（辅助层，前端开发任务时自动触发） |
| 核心行为 | 指导 Agent 按 4 步工作流生成高质量前端 UI 代码 |
| 工作模式 | 布局设计 → 主题设计 → 动画设计 → 代码实现 |
| 支持平台 | Web（桌面端 + 移动端） |
| 核心能力 | 布局规划、主题定制、微交互动画、响应式设计、无障碍支持、组件设计 |

**设计工作流：**
1. **布局设计** — 先用 ASCII 绘制页面结构草图
2. **主题设计** — 定义颜色、字体、间距、阴影
3. **动画设计** — 规划微交互与过渡效果
4. **代码实现** — 生成实际 HTML/CSS/JS 代码

详细规范见 `references/` 目录下各专题文档。

## 触发条件

### 何时应用

- 用户要求生成 UI 组件（按钮、卡片、表单、导航等）
- 用户要求创建落地页、营销页、官网
- 用户要求创建仪表盘、后台管理界面
- 用户要求美化/重构现有界面
- 用户需要前端设计建议（配色、布局、字体等）

### 何时不应用

- 纯后端开发任务（API、数据库、服务器配置等，无 UI 需求）
- 用户明确要求使用特定设计系统（如 Ant Design、Element Plus）且无需通用设计指导
- 用户只需要简单的文本排版（如 Markdown 文档），无需视觉设计

## 工作流概述

### 第一步：布局设计

在编写任何代码之前，先用 ASCII 格式绘制页面布局草图，明确页面结构和内容区域分布。

典型页面结构示例：

```
┌─────────────────────────────────────┐
│         顶部导航栏 / 品牌标识        │
├─────────────────────────────────────┤
│                                     │
│            首屏 Banner              │
│       (标题 + 行动号召按钮)         │
│                                     │
├─────────────────────────────────────┤
│   功能卡片    │  功能卡片  │  功能卡片 │
├─────────────────────────────────────┤
│              页脚                    │
└─────────────────────────────────────┘
```

### 第二步：主题设计

定义统一的视觉语言：
- **色彩**：使用 `oklch()` 定义现代色彩，避免通用 Bootstrap 蓝色（`#007bff`）
- **字体**：从 Google Fonts 中选择（Inter、Outfit、DM Sans 等）
- **间距**：以 0.25rem（4px）为基础单位
- **阴影**： subtle，最多 1-2 层
- **圆角**：现代风格 0.5-1rem，粗野风格 0px

三种预设主题模式见 `references/theme-patterns.md`。

### 第三步：动画设计

使用微语法规划交互效果：

```
按钮按下: 150ms [缩放1→0.95→1]
悬停上浮: 200ms [Y轴0→-2px, 阴影增强]
淡入动画: 400ms 缓出 [Y轴+20→0, 透明度0→1]
滑入动画: 350ms 缓出 [X轴-100→0, 透明度0→1]
弹跳效果: 600ms [缩放0.95→1.05→1]
```

详细规范见 `references/animation-guide.md`。

### 第四步：代码实现

生成实际代码：
- 原型阶段：使用 CDN 引入 Tailwind CSS、Flowbite、Lucide 图标
- 生产阶段：使用 npm + Vite 构建（见 `references/implementation-rules.md`）
- 图片使用 Unsplash 或 placehold.co，禁止虚构图片 URL
- 始终遵循移动端优先的响应式设计

## 注意事项

- **避免通用蓝色**：不要使用 Bootstrap 风格的 `#007bff`，它会显得过时
- **优先 oklch()**：现代浏览器支持 `oklch()` 色彩空间，比 `rgb()`/`hsl()` 更均匀
- **移动端优先**：始终从移动端布局开始，再用 `@media` 扩展到大屏
- **CDN 仅用于原型**：Tailwind/Flowbite CDN 适合快速原型，生产环境请使用 npm + 构建工具
- **语义化 HTML**：使用 `<header>`、`<main>`、`<nav>`、`<section>` 等语义标签
- **对比度达标**：文字与背景对比度至少 4.5:1，确保可读性
- **不要虚构图片 URL**：使用 Unsplash 或 placehold.co 的真实链接
- **动画性能**：优先使用 `transform` 和 `opacity` 实现动画，避免触发重排（layout）
