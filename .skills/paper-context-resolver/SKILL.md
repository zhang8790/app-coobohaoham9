## 能力概述

本技能指导 Agent 在论文复现场景下，通过查阅原始论文来补充 README 和代码仓库未覆盖的关键细节。它是一个辅助层技能，不替代主复现流程，而是在主流程遇到具体缺口时提供针对性证据支持。

| 属性 | 值 |
|------|-----|
| 技能层级 | `helper-tier`（辅助层，通常由编排器调用） |
| 工作模式 | `README-first`（优先信任 README，论文仅作补充） |
| 信息范围 | 窄范围证据补充，非全文摘要 |
| 输出格式 | 结构化文本，明确区分直接证据与推断 |
| 核心能力 | 数据集细节解析、评估协议确认、预处理流程还原、检查点映射澄清、运行时假设识别 |

**能力边界：**
- 不替代主复现流程的编排逻辑
- 不主动触发，需由上层编排器在检测到缺口后调用
- 不提供论文的全文概括，只回答具体复现问题
- 必须明确标注论文证据与 README 之间的冲突

详细方法论见 `references/paper-assisted-reproduction.md`

## 触发条件

### 何时应用（须同时满足）

1. **存在复现关键缺口**：README 和仓库文件在某个复现关键点上存在信息缺失
2. **缺口类型明确**：缺失内容属于数据集版本/划分/预处理、评估协议、检查点映射、运行时假设等复现关键信息
3. **需要窄范围证据**：主流程需要的是针对性的证据补充，而非论文的完整摘要
4. **已有具体问题**：存在一个明确的、具体的复现问题需要回答（如"评测时用的 top-k 是多少"）
5. **已尝试 README-first**：已经先基于 README 和代码进行了复现尝试，发现信息不足以继续

### 何时不应用（满足任一即不应使用）

1. **README 已充分**：README 已经提供了足够的复现细节，无需额外查阅论文
2. **需要论文概述**：用户想要的是对论文内容的总体解释，而非复现支持
3. **意图覆盖 README**：目标是直接推翻 README 的指令，且不打算记录冲突原因
4. **只有论文标题**：唯一可用的输入是论文标题，还没有发现具体的复现缺口

## 输入期望

| 输入项 | 是否必填 | 说明 |
|--------|----------|------|
| 目标仓库元数据 | 是 | 仓库名称、URL、所用框架、任务类型 |
| 复现关键问题 | 是 | 需要填补的具体缺口（如"训练时使用的 batch size 是多少"） |
| 现有 README/仓库证据 | 是 | README 或代码中已有的相关信息，作为对照基准 |
| 已知论文链接 | 否 | arXiv ID、DOI 或论文 URL（如有则直接使用，否则需自行检索） |

---

## 输出期望

| 输出项 | 是否必填 | 说明 |
|--------|----------|------|
| 收窄的来源列表 | 是 | 具体查阅了论文的哪些章节/页面/附录 |
| 复现相关答案 | 是 | 针对具体问题的直接回答 |
| README-论文冲突说明 | 条件必填 | 当论文证据与 README 不一致时，必须明确记录冲突 |
| 证据与推断区分 | 是 | 清楚标注哪些内容来自论文原文（直接证据），哪些是基于原文的合理推断 |

## 注意事项

- **辅助层定位**：本技能为 `helper-tier`，通常应由主编排器检测到复现缺口后调用，不建议 Agent 自行判断触发
- **README 优先原则**：论文证据仅用于补充 README 未覆盖的细节，若论文与 README 冲突，应记录冲突并优先遵循 README 的显式指令（除非有充分理由推翻）
- **窄范围原则**：只回答具体的复现问题，不提供论文的完整方法描述或背景介绍
- **时效性注意**：注意区分论文的 arXiv 版本与正式发表版本，附录和补充材料也需纳入检索范围
- **证据分级**：输出必须区分 `直接证据`（论文原文可明确支撑）与 `合理推断`（基于论文内容的逻辑推导）
- **冲突记录格式**：发现 README-论文冲突时，使用以下格式记录：
  - `README 说法：` ...
  - `论文证据：` ...
  - `冲突点：` ...
  - `建议处理方式：` ...
