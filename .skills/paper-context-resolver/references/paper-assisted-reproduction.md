# 论文辅助复现方法论

本文档为 `paper-context-resolver` 技能的详细操作手册，指导 Agent 如何在论文复现场景下高效、准确地查阅原始论文并提取复现关键信息。

---

## 1. 论文获取策略

### 1.1 确定论文版本

当获得论文链接或 arXiv ID 时，按以下优先级选择查阅版本：

| 优先级 | 版本 | 适用场景 | 注意事项 |
|--------|------|----------|----------|
| 1 | 正式发表版（Conference/Journal） | 已正式发表的论文 | 通常经过审稿修正，方法描述最准确 |
| 2 | arXiv 最新版 | 有多个 arXiv 版本时 | 优先选择 v2/v3 等后续版本，可能包含勘误 |
| 3 | arXiv 初版 | 仅有初版可用时 | 注意可能存在未修正的错误或缺失的实验细节 |

**检查要点：**
- 对比 arXiv 版本与正式版的实验设置章节，确认超参数是否有变动
- 正式版论文可能将部分细节移至补充材料（Supplementary Material），必须同时检索

### 1.2 定位关键章节

根据复现问题的类型，优先查阅以下章节：

| 问题类型 | 优先查阅章节 | 次优先查阅 |
|----------|-------------|-----------|
| 数据集版本/划分/预处理 | 实验设置（Experimental Setup / Datasets） | 附录中的数据预处理细节 |
| 评估协议 | 实验设置、评估指标（Evaluation Metrics） | 结果分析（Results / Analysis） |
| 检查点映射 | 模型架构（Model Architecture）、实现细节 | 附录中的模型配置表 |
| 运行时假设 | 实验设置、训练细节（Training Details） | 代码仓库的 issue 或 README |
| 超参数设置 | 实验设置、附录表格 | 消融实验（Ablation Study） |

## 2. 证据提取工作流

### 2.1 准备工作

在查阅论文前，明确以下信息：
1. 具体要回答的复现问题（一句话描述）
2. README/代码中已有的相关信息（作为对照基准）
3. 仓库所用的框架和任务类型（辅助定位相关章节）

### 2.2 检索步骤

```
Step 1: 定位论文
    - 使用已知的 arXiv ID / DOI / 论文标题检索
    - 确认获取的是最新/最准确的版本

Step 2: 快速定位相关章节
    - 根据问题类型查表定位优先章节
    - 使用 PDF 搜索功能检索关键词
    - 常见关键词：dataset, split, preprocessing, evaluation, metric, checkpoint, hyperparameter, batch size, learning rate

Step 3: 提取直接证据
    - 记录原文中的确切表述（引用原文段落）
    - 记录所在位置（章节名、页码、表格编号）
    - 区分"明确声明"与"隐含推断"

Step 4: 检查补充材料
    - 若正文信息不足，查阅附录和补充材料
    - 补充材料常包含完整超参数表、预处理代码描述等

Step 5: 交叉验证
    - 若论文中存在多个相关描述，检查是否一致
    - 若存在矛盾，记录并说明哪个更可信（通常以实验设置章节的最新描述为准）
```

### 2.3 关键词检索建议

针对不同复现问题，推荐使用以下关键词组合进行 PDF 内搜索：

| 问题类型 | 推荐关键词 |
|----------|-----------|
| 数据集划分 | "train/validation/test split", "dataset split", "random seed" |
| 预处理 | "preprocess", "normalize", "resize", "augmentation", "tokenization" |
| 评估指标 | "evaluation metric", "accuracy", "F1", "BLEU", "mAP", "top-1", "top-5" |
| 训练配置 | "batch size", "learning rate", "optimizer", "scheduler", "epoch", "warmup" |
| 检查点 | "checkpoint", "model weights", "pretrained", "fine-tune", "resume" |
| 运行时 | "hardware", "GPU", "training time", "inference time", "memory" |

## 3. 冲突记录规范

当论文中的证据与 README/仓库中的描述不一致时，必须按以下格式记录：

```markdown
### 冲突记录：[冲突简述]

- **README 说法**：
  [直接引用 README 或代码中的相关描述]

- **论文证据**：
  [直接引用论文原文，标注章节/页码/表格]

- **冲突点**：
  [用一句话说明核心分歧]

- **建议处理方式**：
  [优先遵循 README / 优先遵循论文 / 需要人工确认，并说明理由]
```

**处理优先级原则：**
1. 若 README 是官方维护且明确声明了某个设置，优先遵循 README
2. 若 README 含糊不清而论文明确声明，优先遵循论文并记录冲突
3. 若两者都无明确说明，基于论文进行合理推断并标注为"推断"
4. 若冲突涉及核心复现结果（如评估指标定义），建议标记为"需要人工确认"

## 4. 输出模板

每次调用本技能后，Agent 应产出以下结构化输出：

```markdown
## 复现问题
[一句话描述本次需要回答的具体复现问题]

## 查阅来源
| 来源 | 位置 | 说明 |
|------|------|------|
| [论文标题] | 第 X 页，Y 章节 | [简述该位置包含的信息] |
| [补充材料标题] | 附录 Z | [简述该位置包含的信息] |

## 复现答案
[针对具体问题的直接回答，尽量简洁]

### 证据分级
- **直接证据**：[列出可直接支撑答案的论文原文或数据]
- **合理推断**：[列出基于论文内容逻辑推导出的结论]

## 冲突记录
[若存在 README-论文冲突，按第 3 节格式记录；若无冲突，写"无冲突"]

## 置信度评估
- [ ] 高置信度：论文中有明确声明直接回答此问题
- [ ] 中置信度：论文中有相关信息，但需要一定推断
- [ ] 低置信度：论文中信息不足，答案基于间接推断或通用实践
```

## 5. 质量准则

### 5.1 准确性原则

- **原文优先**：优先引用论文原文，避免用自己的话重新解释导致偏差
- **位置标注**：每个证据都必须标注来源位置（章节/页码/表格），方便人工复核
- **版本一致**：确保引用的论文版本与复现目标一致（arXiv 初版 vs 正式版可能不同）

### 5.2 反幻觉规则

- **禁止编造**：若论文中找不到相关信息，明确回答"论文中未明确提及"，不可猜测
- **区分等级**：必须明确区分"直接证据"、"合理推断"和"无法确认"
- **避免过度概括**：基于论文中某数据集的设置，不可推断另一数据集的设置，除非论文明确声明

### 5.3 效率原则

- **精准检索**：不要通读全文，使用关键词快速定位相关章节
- **按需深入**：若正文章节已提供足够信息，无需继续查阅附录
- **及时终止**：若论文确实未覆盖该问题，及时终止并报告"信息缺失"

### 5.4 常见陷阱

| 陷阱 | 说明 | 应对方式 |
|------|------|----------|
| 混淆训练集与测试集预处理 | 论文可能只对训练集做数据增强 | 分别确认 train/val/test 的预处理流程 |
| 忽略默认参数 | 论文可能只列出非默认参数 | 确认未提及的参数是否使用框架默认值 |
| 版本差异 | arXiv 版与正式版可能有不同设置 | 明确说明引用的是哪个版本 |
| 补充材料遗漏 | 关键细节可能在补充材料中 | 正文信息不足时必须检查补充材料 |
| 隐式假设 | 论文可能假设读者已知某些设置 | 基于领域常识推断时必须标注为"推断" |
