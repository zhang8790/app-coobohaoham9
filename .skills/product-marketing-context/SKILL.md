## 能力概述

本技能帮助用户创建并维护一个产品营销上下文文档，存储于 `.agents/product-marketing-context.md`。该文档捕获产品定位、目标受众、用户画像、痛点、竞品格局、差异化、异议处理和品牌声音等基础信息，其他营销技能可直接引用，避免用户重复描述。

| 属性 | 值 |
|------|-----|
| 技能层级 | `helper-tier`（辅助层，可被编排器调用或用户直接触发） |
| 核心行为 | 创建/更新 `.agents/product-marketing-context.md` 文件 |
| 工作模式 | 支持「自动草稿」和「对话引导」两种设置路径 |
| 文档用途 | 供其他营销技能引用，消除重复描述 |
| 核心能力 | 产品概述、目标受众、用户画像、痛点分析、竞品分析、差异化定位、异议处理、品牌声音、客户语言、证明点、业务目标 |

**设置路径：**
- **自动草稿（推荐）**：研究代码库中的 README、落地页、营销文案、package.json 等，自动生成 V1 草稿，用户审阅修正
- **从零开始**：对话式逐节收集信息，一次一个章节

详细工作流和 12 节模板见 `references/product-marketing-context-guide.md`

## 工作流概述

### Step 1: 检查现有上下文

首先检查 `.agents/product-marketing-context.md` 是否已存在。同时检查 `.claude/product-marketing-context.md`（旧版路径）——如果在 `.claude/` 找到但 `.agents/` 没有，询问用户是否迁移。

- 若已存在：读取内容并总结已捕获的信息，询问用户需要更新哪些章节
- 若不存在：提供两种选项（自动草稿 或 从零开始），大多数用户偏好自动草稿

### Step 2: 收集信息

**自动草稿路径：**
1. 读取代码库：README、落地页、营销文案、关于页面、meta 描述、package.json、现有文档
2. 基于发现的内容起草所有章节
3. 呈现草稿并询问："哪些需要修正？哪些缺失了？"
4. 迭代直到用户满意

**从零开始路径：**
- 逐节对话式收集，一次只处理一个章节
- 每节先简要说明要捕获什么，再提问，确认准确性，再进入下一节
- 强调捕获客户的**原话**——精确短语比润色后的描述更有价值

### Step 3: 创建文档

收集完信息后，在 `.agents/product-marketing-context.md` 创建文档，包含 12 个标准章节（见参考文档）。使用 `write_file` 工具直接写入文件。

### Step 4: 确认并保存

- 展示完成的文档内容
- 询问是否需要调整
- 保存到 `.agents/product-marketing-context.md`
- 告知用户："其他营销技能现在会自动引用此上下文。随时运行 /product-marketing-context 来更新它。"

## 输入期望

| 输入项 | 是否必填 | 说明 |
|--------|----------|------|
| 工作空间路径 | 是 | 要分析的代码库/项目路径（用于自动草稿） |
| 设置路径偏好 | 否 | `自动草稿`（默认）或 `从零开始` |
| 现有上下文路径 | 否 | 旧版 `.claude/product-marketing-context.md` 迁移检查 |
| 用户修正意见 | 是（V1 阶段） | 对自动草稿的反馈或对话中的回答 |
| 要更新的章节 | 条件必填 | 上下文已存在时，用户指定更新哪些章节 |

## 输出期望

| 输出项 | 是否必填 | 说明 |
|--------|----------|------|
| `.agents/product-marketing-context.md` | 是 | 结构化产品营销上下文文档（通过 write_file 创建） |
| 章节摘要 | 是 | 每个已捕获章节的简要总结，供用户验证 |
| 迭代更新 | 条件必填 | 根据用户反馈更新文档内容 |
| 迁移提示 | 条件必填 | 发现旧版 `.claude/` 路径时提供迁移建议 |

## 注意事项

- **核心行为是创建文件**：本技能必须实际调用文件写入工具（如 `write_file`）在用户工作空间中创建 `.agents/product-marketing-context.md`，不能只输出文本到对话中
- **自动草稿只是起点**：自动生成的 V1 草稿需要用户审阅和修正，不可直接视为最终版本
- **原话优先**：询问客户如何描述问题和解决方案时，优先捕获精确短语（原话），而非润色后的描述。例如问"是什么让他们决定换用你们？"而非"你们解决了什么问题？"
- **验证每一节**：总结每个章节的内容并请用户确认后再进入下一节
- **按需跳过**：不是所有产品都需要全部 12 个章节（例如 B2C 产品可能不需要用户画像，简单工具可能不需要复杂的竞品分析）
- **具体提问**：问"让他们转向你们的头号挫败感是什么？"而不是"你们解决了什么问题？"
- **要求示例**："能给我举个例子吗？"通常能引出更好的答案
- **上下文引用**：告知用户其他营销技能会自动引用此文档，运行 `/product-marketing-context` 可随时更新
