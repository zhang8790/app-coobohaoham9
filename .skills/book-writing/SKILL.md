## 能力概述

本技能帮助用户系统化地完成一本完整书籍的策划、起草和修订，确保叙事连贯性和作者声音一致性。

| 属性 | 值 |
|------|-----|
| 技能层级 | `helper-tier`（辅助层，写作任务时触发） |
| 核心行为 | 创建/维护 `~/book-writing/` 目录下的项目记忆文件 |
| 工作模式 | 设置 → 蓝图设计 → 逐章起草 → 分阶段修订 |
| 适用体裁 | 非虚构、小说、混合体裁 |
| 核心能力 | 书籍策划、章节架构、声音一致性、修订工作流、进度管理 |

**工作空间架构：**

本技能在用户工作空间中维护以下目录结构：

```
~/book-writing/
├── memory.md          # 热数据：状态、声音、书稿状态、下一步行动
├── chapters/          # 温数据：章节笔记和草稿检查点
├── revisions/         # 温数据：逐轮修订日志
└── archive/           # 冷数据：已废弃的方向和过时的提纲
```

详细规范见各 references/ 文档。

## 触发条件

### 何时应用

- 用户说"我要写一本书"或"帮我规划一本书"
- 用户需要书籍结构、章节大纲或内容策划
- 用户正在起草章节内容，需要写作支持
- 用户需要修订书稿，进行结构调整或润色
- 用户需要管理多章节的写作进度
- 用户遇到写作瓶颈，需要突破建议

### 何时不应用

- 用户只需要写短文（博客、邮件、社交媒体）
- 用户明确要求使用其他写作工具或流程
- 用户的问题与书籍写作无关

## 工作流概述

### 第一步：设置与激活

首次使用时，读取 `references/setup-guide.md` 初始化本地记忆：
- 创建 `~/book-writing/memory.md`
- 捕获书籍类型、目标读者、核心承诺
- 确定用户当前阶段和即时优先级

### 第二步：蓝图设计

使用 `references/blueprint.md` 设计书籍蓝图：
- 确定书籍承诺（读者、问题、转变、边界）
- 定位选择（类别、差异化、可信度来源）
- 结构骨架（开头钩子、核心弧线、中段加速、结尾 payoff）
- 章节地图（每章的一句话目的、读者成果、素材清单）

### 第三步：逐章起草

使用 `references/chapter-loop.md` 逐章起草：
- 定义章节契约（成果、输入、边界）
- 单向起草（从头到尾，不做微观编辑）
- 验证连贯性（开头匹配承诺、每节有价值、主线清晰、结尾自然过渡）
- 记录决策（范围变更、声音调整、新风险）

### 第四步：分阶段修订

使用 `references/revision-rubric.md` 进行四轮修订：
1. **结构完整性** — 章节顺序、合并冗余、插入过渡
2. **论点与叙事强度** — 证据支撑、故事节拍、消除矛盾
3. **清晰度与可读性** — 每段一个核心思想、简化复杂段落、替换术语
4. **行文质量** — 动词驱动、删减重复、声音一致

## 注意事项

- **核心行为是创建文件**：本技能必须实际调用文件写入工具（如 `write_file`）在用户工作空间中创建 `~/book-writing/memory.md` 和相关文件
- **锁定书籍承诺后再起草**：在明确受众、核心承诺、转变范围和边界之前，不要生成大段文本
- **保持活的蓝图**：每当方向变化时更新蓝图，不要让它过时
- **按成果而非字数写作**：每章必须交付一个具体的读者成果，不为凑字数而写
- **保护声音一致性**：在记忆中跟踪声音特征，跨章节保持一致的视角、时态、阅读水平和句式节奏
- **分阶段修订**：结构、论点、清晰度、润色四个阶段分开进行，不要混合
- **尽早暴露风险**：发现逻辑薄弱、冗余章节、未兑现承诺和节奏空洞时立即标记
- **每次会话结束于最小下一步**：每次交互后留下用户可以立即执行的具体下一步
- **安全与隐私**：项目记忆存储在用户本地 `~/book-writing/`，不发送到手稿数据到外部 API，不访问 `~/book-writing/` 以外的文件进行记忆存储
