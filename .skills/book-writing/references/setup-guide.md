# 设置指南

首次激活本技能时使用，或当 `~/book-writing/memory.md` 缺失时使用。

## 激活首选项

首先说明你可以为长篇幅手稿保持本地项目记忆，然后询问用户是否希望此工作流在每次请求写书帮助时自动激活。

将以下激活偏好存储在 `~/book-writing/memory.md` 中：

- **激活范围**：总是为书籍工作自动激活，还是仅在请求时激活。
- **主动性级别**：主动建议、平衡模式、仅按需响应。
- **会话风格**：战略规划优先、起草优先、还是修订优先。

如果用户拒绝设置，继续以零摩擦的方式立即提供帮助，并将集成状态设置为 `declined`。

## 最小化基线收集

以自然对话方式收集最小但有用的基线信息，一次一个高价值问题：

- **书籍类型**：非虚构、小说，还是混合体裁？
- **目标读者和承诺成果**：为谁而写，读完能获得什么？
- **当前阶段**：构思、大纲、起草、修订，还是接近完成？
- **即时优先级**：今天必须推进什么？

不要运行冗长的问卷。一次只问一个高价值问题。

## 首次会话交付物

在结束设置前，必须产出至少一个实用成果：

- 一个简洁的书籍承诺句。
- 一份章节地图草稿。
- 一个聚焦的章节简报。
- 对当前书稿部分的修订计划。

## 持久化规则

启用本地记忆时：

- 从 `memory-template.md` 创建 `~/book-writing/memory.md`。
- 每次有意义的会话后更新 `last` 时间戳。
- 保持摘要简短且以决策为导向。
- 绝不存储秘密、凭证或无关的个人数据。
